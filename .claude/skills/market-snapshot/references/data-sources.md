# 데이터 소스 가이드 — 시장 자동 감지 + 무료 1차 자료

분석가 4명·리서처·트레이더·market-data-engineer가 공통으로 사용하는 데이터 수집 규칙.

## 0. 단일 진실 소스 (SSOT) — 스냅샷 우선 [최우선]

`_workspace/00_market_snapshot.json`이 존재하면, 그 안에 고정된 값은 **단일 진실 소스**다. 모든 에이전트는 다음 항목을 스냅샷 값으로만 사용한다(직접 찾은 다른 값으로 대체 금지):

- 주가(`price`), 시가총액(`market_cap`), 발행주식수(`shares_outstanding`)
- 환율(`fx`), 벤치마크 지수 레벨(`benchmark`)
- 재무 핵심값(`financials`), 밸류에이션 배수(`valuation`)
- 기술지표는 `00_indicators.json`이 SSOT다 — 다시 계산하지 않는다.

스냅샷에 **없는** 항목(세그먼트별 매출, 정성 정보, 뉴스, 여론 등)만 직접 수집한다. `confidence: "unavailable"` 필드는 추측으로 메우지 말고 미확보로 다룬다.

**스냅샷 오류 발견 시:** 어느 에이전트든 스냅샷 값이 1차 출처와 명백히 다른 것을 발견하면, 자기 보고서 상단에 `⛔스냅샷 오류`(field/잘못된 값/올바른 값/출처) 플래그를 남긴다. 오케스트레이터가 market-data-engineer를 재호출해 스냅샷을 갱신하고 하류를 재실행한다.

## 1. 시장 자동 감지

| 단서 | 판정 | 예시 |
|------|------|------|
| 알파벳 티커 (1~5자), NYSE/NASDAQ 언급 | **미국** | AAPL, NVDA, TSLA, MSFT |
| 6자리 숫자 코드, KOSPI/KOSDAQ, 한글 회사명 | **한국** | 005930(삼성전자), 035720(카카오) |
| 영문 회사명이지만 한국 기업 | **한국** | Samsung Electronics, Hyundai |
| 판별 모호 | 웹 검색으로 상장 거래소 확인 후 결정 | - |

## 2. 미국 주식 데이터 소스

- **SEC EDGAR** (1차, 무료): 10-K(연간), 10-Q(분기), 8-K(수시). `database-lookup` 스킬이 SEC EDGAR API를 지원한다. 재무제표·리스크 요인(Item 1A)의 원천.
- **시세·기술 데이터**: 번들 스크립트(`collect_market_data.py`, yfinance 기반)가 1차. 웹 검색으로 교차 확인.
- **뉴스·촉매**: WebSearch로 실적 발표, 가이던스, 애널리스트 목표가, 규제·소송.
- **거시**: `database-lookup`의 FRED(금리·물가·환율).

## 3. 한국 주식 데이터 소스

- **DART 전자공시** (1차, 무료): 사업·분기·반기보고서, 주요사항보고서. `k-dart` 스킬(OpenAPI) 우선, 또는 `https://dart.fss.or.kr`.
- **시세·기술 데이터**: 번들 스크립트(.KS/.KQ 자동 시도)가 1차. 네이버 증권·KRX 정보데이터시스템으로 교차. 차단 시 `insane-search`.
- **종목 기본 정보**: `korean-stock-search` 스킬.
- **뉴스·촉매**: `naver-news-search` 스킬, WebSearch. 증권사 리포트 요지는 `daishin-report-search`.
- **거시**: 한국은행 기준금리, 원/달러 환율, 업종 사이클.

## 4. 접근 차단 시

1. `insane-search` 스킬로 우회 (네이버·X·레딧·차단 사이트에 강함)
2. 그래도 안 되면 대체 출처를 찾는다
3. 끝내 못 구하면 **"데이터 미확보"** / **"검증불가(접근차단)"** 로 명시하고 진행한다. 추측으로 채우지 않는다.

## 5. 출처 인용 규칙 (전 에이전트 공통)

- 모든 수치·사실 주장에 출처와 **시점**을 붙인다: "FY2026 Q1 매출 $44.1B (10-Q, 2026-05 제출)".
- 1차 자료(공시) > 회사 IR > 평판 좋은 언론 > 블로그·커뮤니티 순으로 신뢰한다.
- 오래된 데이터를 현재처럼 쓰지 않는다.
- 추정·해석은 사실과 분리해 "[추정]" / "[해석]" 태그로 구분한다.
