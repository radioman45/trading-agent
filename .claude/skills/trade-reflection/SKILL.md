# Trade Reflection — 의사결정 복기

이 하네스가 한 번의 분석 도구를 넘어 **학습하는 시스템**이 되는 메커니즘이다. 결정 당시의 논거·확신도·트리거와 실제 전개를 대조해, 다음 결정을 바꿀 교훈만 추출한다.

## 핵심 원칙: 결과론 금지

좋은 결정이 나쁜 결과를 낳을 수 있고, 나쁜 결정이 운 좋게 수익을 낼 수 있다. 평가 대상은 **결과가 아니라 당시 정보 기준의 의사결정 품질**이다:

- 수익 났지만 손절 규칙을 어겼다 → **나쁜 결정** (다음엔 그 운이 없다)
- 손실 났지만 무효화 조건 발동 즉시 손절했다 → **좋은 결정** (시스템이 작동했다)

## 복기 절차

1. `decisions/journal.md`에서 대상 결정을 찾는다 (사용자가 특정 안 하면 `결과:` 미기록 항목을 보여주고 확인).
2. 사용자가 가져온 결과(현재가/청산가, 그간의 전개)를 정리한다. 가능하면 스크립트로 현재 시세를 확인한다:
   ```bash
   python .claude/skills/market-snapshot/scripts/collect_market_data.py --ticker {티커} --out-dir _workspace/_reflection
   ```
   같은 기간 **벤치마크 등락도 확인해 알파(벤치마크 대비 초과수익)를 산출한다** — 당시 결정의 스냅샷 `benchmark.primary` 레벨과 현재 레벨을 대조하거나, 벤치마크 티커로 스크립트를 한 번 더 실행한다. 절대 수익률만으로 평가하지 않는다: 시장 전체가 오른 장의 +10%는 실력이 아닐 수 있고, 폭락장의 -5%는 잘 방어한 것일 수 있다.
3. **4축 대조:**
   | 축 | 질문 |
   |----|------|
   | 논거 적중 | 핵심 논거가 실제로 전개됐는가, 틀렸다면 정보 부족이었나 추론 오류였나 |
   | 무효화 작동 | 무효화 조건·철회 트리거가 발동했는가, 발동 시 따랐는가(따랐어야 했는가) |
   | 리스크 설계 | 손절 폭·비중이 실제 변동성과 맞았는가 (너무 좁아 털렸나, 과대 비중이었나) |
   | 누락 변수 | 당시 파이프라인이 못 본 것은 무엇인가, 어느 에이전트의 사각지대였나 |
4. `decisions/journal.md`의 해당 항목 `결과:` 줄을 갱신한다 (이 줄만 수정 — 나머지 기록은 보존). 형식: `결과: [복기 {YYYY-MM-DD}] {±X}%, 벤치마크 {±Y}% (알파 {±Z}%p) — {의사결정 품질 1줄 평가}`. 파이프라인이 남긴 `[중간점검 ...]` 줄이 있으면 복기 확정으로 덮어쓴다.
5. 교훈을 추출해 `decisions/lessons.md`에 append한다.

## 교훈 추출 기준 — 일반화 가능한 것만

- **교훈인 것:** 패턴 — "이벤트 D-3 이내 일괄 진입은 갭에 취약했다", "심리 극단 낙관에서의 신규 매수는 2회 연속 고점 진입이었다", "ATR 1.5배 미만 손절은 정상 변동에 털렸다"
- **교훈이 아닌 것:** 1회성 불운("갑자기 CEO가 사임했다"), 결과론("올랐으니 더 살 걸"), 종목 고유 사건의 과잉 일반화
- 같은 패턴이 2회 이상 관찰되면 교훈의 확신도를 올린다. 1회 관찰은 "가설" 등급으로 적는다.

## 교훈 포맷 (`decisions/lessons.md`에 append)

```markdown
---
## L-{번호}: {교훈 제목 1줄}
- 등급: [가설(1회 관찰) | 교훈(2회+) ]
- 적용 대상: [bull/bear | research-manager | trader | risk | portfolio-manager | 전체]
- 내용: {다음 결정에서 무엇을 다르게 해야 하는가 — 행동 가능한 형태}
- 근거 사례: {journal의 날짜·종목}
```

`적용 대상`이 중요하다 — 각 에이전트는 시작 시 자기에게 적용되는 교훈만 골라 반영한다.

## 안티패턴

- 교훈 남발: 모든 복기에서 교훈을 만들 필요 없다. "시스템이 설계대로 작동했고 결과는 확률의 영역"이 결론이면 그게 정답이다.
- 기존 교훈과 모순되는 새 교훈이 생기면 둘 다 보존하고 모순을 명시한다 — 다음 복기가 판정한다. 기존 교훈 삭제는 같은 패턴의 반례가 2회 이상일 때만.
