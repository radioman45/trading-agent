# Risk Gate — 리스크 토론 + 최종 게이트

좋은 거래 계획도 리스크 관점에 따라 다르게 보인다. 세 성향이 같은 계획을 독립적으로 공격하고, 포트폴리오 매니저가 그 상충을 중재해 최종 판정한다. 이 구조가 단일 리스크 검토보다 나은 이유: 한 명의 검토자는 자기 성향(대개 보수)의 사각지대를 못 본다 — 기회비용도 리스크다.

## Part A: 리스크 토론자 평가 (3성향 공용)

### 평가 축 (모든 성향 공통, 우선순위만 다름)

| 축 | 점검 내용 |
|----|----------|
| ① 산식 검산 | 포지션 역산(R% ÷ 손절거리), **꼬리 검산(비중 × 꼬리 손실률 ≤ 계좌 2%)**, 손익비, 손절 폭 vs ATR — **직접 재계산** |
| ② 변동성 적합성 | 손절·진입이 이 종목의 변동성(ATR%·갭 빈도)을 견디는가 |
| ③ 이벤트 리스크 | 캘린더상 고변동 이벤트와 진입 타이밍의 관계 |
| ④ 쏠림·유동성 | 심리 보고서의 쏠림(과열 진입인가), 거래량 대비 포지션 규모 |
| ⑤ 시나리오 비대칭 | 상방/하방 시나리오가 균형 있게 반영됐는가 (성향별로 과소평가된 쪽 지적) |
| ⑥ 확신도 정합성 | 리서치 확신도와 포지션 공격성이 비례하는가 |
| ⑦ 바벨·상관·레짐 | 거시 레짐 정합성, 바벨 다리 태깅, 상관 연쇄, 현금·레버리지 규율 — `references/barbell-correlation.md` 참조 |

축 ⑦ 상세(`references/barbell-correlation.md`를 읽고 적용):
- **거시 레짐 정합:** `00_macro_regime.md`의 레짐·바벨 기울기와 이 거래 방향이 맞는가. risk-off/전환점 레짐에서 공격적 진입이면 가점 검증.
- **바벨 다리:** 종목이 공격(AI/성장) vs 방어(원자재/현금흐름) 어느 다리인지, 현 레짐에 순풍/역풍인지. **크립토는 공격 다리로 태깅한다** — '디지털 골드' 서사와 무관하게 위기 시 상관이 주식·성장 자산과 수렴한다(방어 다리 취급 금지, `references/barbell-correlation.md` §3).
- **상관 연쇄:** 시장 급락 시 헤지·방어 가정이 깨지는가(골드 마진콜형 연쇄). 기존 노출과 같은 거시 변수에 집중되는가.
- **현금·레버리지 규율:** 일괄·고레버리지 vs 단계별·현금 여유. 환율 익스포저(한국 투자자).

### 산출 템플릿 (`05_risk_{성향}.md`)

```markdown
# 리스크 평가 ({공격|중립|보수}) — {회사명}
## 종합 평가 (3줄 이내)
{이 성향의 렌즈로 본 계획의 상태}
## 축별 지적
### ①~⑥ 중 문제 있는 축만
- 지적: {구체적 문제}
- 대안: {수정안 — 숫자까지. "비중 12%→8%, 근거: ..."}
- 심각도: [⛔치명 | ⚠️중대 | ℹ️경미]
## 성향 권고: [승인 | 조건부 승인(조건 명시) | 거부]
```

심각도 기준: ⛔치명 = 이대로 집행하면 구조적 손실 위험(손절 누락, 비중 상한 초과, 산식 오류, 방향 불일치). ⚠️중대 = 수익성·일관성을 해침. ℹ️경미 = 개선 여지.

### 성향별 우선 렌즈

- **공격:** 기회비용을 리스크로 본다. 과소 비중·과도 분할·소심한 목표가를 지적한다. 레짐이 risk-on·바벨 순풍이면 과소 노출도 리스크다. 단, 산식 위반을 "공격적이니 OK"로 눈감지 않는다 — 상한은 상한이다.
- **중립:** 가정의 현실성을 본다. 시나리오 확률·손익비 산식의 근거·양 극단 성향이 놓친 중간 지대·상관 가정을 지적한다.
- **보수:** 최악 생존을 본다. 갭 리스크·이벤트 직전 진입·쏠림 과열·하방 과소평가·상관 연쇄(헤지가 위기에 깨짐)·디플레/레짐 역풍을 지적한다. 단, 모든 거래에 반대하는 것은 평가가 아니다 — 견고하면 견고하다고 쓴다.

## Part B: 포트폴리오 매니저 최종 게이트

### 게이트 체크리스트 (재분석 아님 — 점검만)

- [ ] **검증·포트폴리오 모드 판정 (요약 박스 입력 — 최상단에서 먼저):** ① 검증 모드 — `00_factcheck.md`(있으면)에 "미확인" 강등이 있거나, `00_indicators.json`의 `cross_check.status`가 `mismatch`(이중 소스 종가 상충 — 원인 미해소)면 `DEGRADED_DATA`. `cross_check`가 미수행(`skipped_primary_is_fallback`·`skipped_error`·`skipped_kr`·`skipped_crypto_quote`)인데 스냅샷에 독립 웹 교차 기록(`data_quality.conflicts`/`notes` 또는 `price.source`의 2차 출처 — KR은 네이버·KRX, 비달러 쿼트 크립토는 업비트·빗썸 대조)도 없으면 역시 `DEGRADED_DATA`(단일 소스 미검증 상태를 "완전 검증"으로 표기 금지). 어느 쪽도 아니면 `FULLY_VERIFIED`. 강등 수치가 결론을 떠받치면(load-bearing) 일반 APPROVE 금지 — 해당 수치 확인을 조건화(CONDITIONAL_APPROVE)하거나 REVISE. **load-bearing 판정 기준(고정):** 그 수치를 "미확인"으로 간주했을 때 채택 계획의 방향·비중·손절 중 하나라도 바뀌면 load-bearing이다 — 셋 다 그대로면 아니다. ② 포트폴리오 모드 — `decisions/portfolio.json` 상태로 판정(`PORTFOLIO_AWARE` | `SINGLE_TRADE_ONLY` | `PORTFOLIO_STALE` — 기준은 portfolio-risk 스킬). **holdings가 비어 있으면 결정문에 "포트폴리오를 반영했다"류 표현 금지** — 실제로는 단일 캠페인 관점만 봤기 때문이다
- [ ] **방향 정합성:** 거래 계획 ↔ 리서치 계획 방향 일치
- [ ] **리스크 한도 (이중):** ① 손절 기준 1회 손실 ≤ 계좌 2% **그리고** ② 꼬리 기준 비중 × 꼬리 시나리오 손실률 ≤ 계좌 2%. 성향별 비중 상한 준수(초과 채택 금지 — 저변동 인컴 예외는 꼬리 검산 통과 + 명시 정당화 시에만 상한 2배까지), 손익비 ≥ 1.5 (또는 정당한 보류)
- [ ] **포트폴리오 영향:** `05_portfolio_impact.md`(있으면) — 편입 후 단일 종목/단일 거시 변수 집중 🚩, 동시 꼬리 손실 합, 바벨 균형이 레짐 권고와 정합하는가. 🚩 항목을 무시하고 승인하려면 명시 정당화. 보유 미등록이면 "단일 종목 관점만 — 보유 등록 권고"를 결정문에 명시
- [ ] **⛔치명 지적 처리:** 리스크 토론의 ⛔치명이 하나라도 있으면 APPROVE 불가 → REVISE 또는 REJECT
- [ ] **상충 중재:** 세 성향의 상충 지적에 채택안 + 이유 명시 (기계적 평균 금지)
- [ ] **거시 레짐·바벨 정합:** `00_macro_regime.md`(있으면)의 레짐·바벨 기울기와 이 거래가 정합하는가. 레짐 역풍(risk-off에 공격 진입)이면 그 사유를 명시적으로 정당화. 상관 연쇄·현금 규율 점검(`references/barbell-correlation.md`)
- [ ] **역발상 점검:** `.claude/skills/contrarian-check/SKILL.md`로 채택 계획이 "모두가 아는 트레이드"인지, 쏠림·레버리지 연쇄 리스크가 있는지 점검. 확신이 과하면 비중을 줄인다 — 단 역발상 점검은 방향 중립이다(하향 전용 아님)
- [ ] **기회비용 게이트 (관망·축소 판정 시 의무):** 관망/보류/비중 축소로 기울면 "관망의 기대 비용"(강세 시나리오 확률 × 상승폭)을 명시하고, L0가 risk-on인데 종목 논거(재무·추세)가 생존해 있으면 관망 선택에 초과 근거를 요구한다. 기회비용도 손실이다 — 자본 보존 편향으로 강세장을 통째로 놓친 것이 이 시스템의 확인된 실패 패턴이다(lessons 삼성 사례: 약세 편향으로 +18.5% 중 절반만 포착). 신중을 반복 보상하는 다단계 구조의 관성을 게이트가 상쇄한다
- [ ] **교훈 위반:** `decisions/lessons.md`의 교훈과 정면 충돌하는 결정이면 그 사유를 명시적으로 정당화 (모르고 위반 금지)

### 판정 규칙

| 판정 | 조건 | 후속 |
|------|------|------|
| **APPROVE** | ⛔치명 0 + 체크리스트 전부 통과 + **부가 조건 없음** + 검증 모드 FULLY_VERIFIED | 최종 거래 지침 확정, 저널 기록 |
| **CONDITIONAL_APPROVE** | ⛔치명 0이나 ⚠️경고를 실행 조건으로 흡수(조건부 승인), 또는 REVISE 해소 후 승인, 또는 DEGRADED_DATA에서 강등 수치 확인을 조건화한 승인 | 조건마다 **원 경고 ID를 매핑**해 결정문 경고 대장에 보존(경고가 조건 속으로 사라지지 않게). 저널 기록 — pending 결산·복기·캘리브레이션에서는 APPROVE와 동일하게 최종 판정 1건으로 취급 |
| **REVISE** | ⛔치명 있으나 1회 수정으로 해소 가능 | 트레이더(계획 문제) 또는 리서치 매니저(방향 문제)에게 구체 지시 — **1회만**, 비중이 바뀌면 portfolio-risk-analyst의 편입 후 재계산(`05_portfolio_impact.md` 갱신)을 거쳐 재점검, 재제출 미해소 시 REJECT. 저널 기록(중간 판정 — 아래 저널 포맷의 REVISE 규칙). 해소 후 승인은 조건 유무에 따라 APPROVE/CONDITIONAL_APPROVE — **이때 결정문에 `REVISE 해소: {지시 요지} → {해소 확인}` 줄을 의무로 남긴다**(리스크 토론 3편은 재작성되지 않아 ⛔치명 표기가 파일에 잔존하므로, 이 줄이 doctor·독자가 "잔존 ⛔ = 해소됨"을 판별하는 기계판독 마커다) |
| **REJECT** | 손익비 미달 억지 계획, 논거 빈약, 교훈 정면 위반, REVISE 미해소 | 거부 사유 + 재고 조건 기록, 저널 기록 (거부도 의사결정이다) |

조건부 승인을 일반 APPROVE로 적지 않는다 — "조건을 붙였다"는 사실 자체가 리스크 정보이고, 라벨이 그것을 숨기면 사용자는 완전 검증된 결정으로 오독한다.

### 결정문 템플릿 (`06_final_decision.md`)

```markdown
# 최종 결정 — {회사명} ({티커})

## 판정 요약 박스 (첫 화면 — 이 5줄로 "얼마나 검증된 결정인지"가 즉시 보여야 한다)
| 판정 | [APPROVE | CONDITIONAL_APPROVE(조건 {n}건) | REVISE | REJECT] |
| 검증 모드 | [FULLY_VERIFIED | DEGRADED_DATA — 강등 항목 1줄] |
| 포트폴리오 모드 | [PORTFOLIO_AWARE | SINGLE_TRADE_ONLY(보유 미등록) | PORTFOLIO_STALE] |
| 경고 집계 | ⛔ {n} / ⚠️ {n} — 조건화 {n}·기각 {n} (경고 대장 참조) |
| 데이터 기준 | {스냅샷 as_of + 세션 앵커} |

- 판정: [APPROVE | CONDITIONAL_APPROVE | REVISE | REJECT] | 시점: {YYYY-MM-DD}

## 실행 카드 (5줄 — 사용자가 이것만 보고 행동할 수 있어야 한다)
| 방향/판정 | {매수·관망·매도 + 판정} |
| 진입 | {가격대·분할} |
| 비중 | {성향별 %} |
| 손절/무효화 | {가격 + 조건 1줄} |
| 다음 점검 | {날짜 또는 트리거} |

## 경고 대장 (경고 → 조건 매핑 — 흡수된 경고는 사라지지 않는다)
| ID | 출처(성향/축) | 심각도 | 요지 | 처리 | 조건 번호 / 기각 사유 |
|----|--------------|--------|------|------|---------------------|
| W1 | {보수/②} | ⚠️ | {손절 트리거 노이즈 근접} | 조건화 | 조건 B(시간 기반 보조 손절) |
| W2 | {공격/⑤} | ⚠️ | {시드 상향 요구} | 기각 | {왜 REVISE하지 않았는지 — 근거의 질 판정 1줄} |
⚠️중대 이상은 전부 이 대장에 올린다. ID(W1, W2 …)는 **PM이 대장 작성 시 발번**한다 — 리스크 토론자 산출물에는 ID가 없으므로 출처(성향/축)를 함께 적어 역추적을 보장한다. 기각에는 "왜 REVISE하지 않았는지"를 반드시 적는다 — 여러 독립 경고가 같은 방향을 가리키는데 전부 조건으로 흡수되고 있으면 그 자체가 과신 신호다.

## 판정 근거
{체크리스트 결과 + 세 성향 상충의 중재 내용 — 이하 전부는 감사 추적용 상세다}
## 최종 거래 지침 (APPROVE/CONDITIONAL_APPROVE 시)
{채택된 성향별 계획 — 수정 반영 최종본: 진입/손절/비중/목표/지평 (+ 조건 목록)}
## 모니터링 트리거
{철회 트리거 + 재평가 시점}
## 잔여 리스크 (수용한 것)
{알고도 수용한 리스크 — 복기 때 대조할 것}
```

### 의사결정 저널 포맷 (`decisions/journal.md`에 append)

```markdown
---
## {YYYY-MM-DD} {회사명}({티커}) — {판정}
- 방향/확신도: {매수|관망|매도} {N}% ({확신도 정의 — 표준 앵커: "12개월 내 벤치마크 대비 초과수익 확률". 다른 정의가 필요하면 병기})
- 핵심 논거: {1줄 × 최대 3}
- 채택 계획: 진입 {} / 손절 {} / 비중 {} / 목표 {} / 지평 {}
- 무효화 조건: {}
- 잔여 리스크: {}
- 기회비용 메모: {관망·축소 판정 시 — 포기한 강세 시나리오의 확률×상승폭 1줄. 진입 판정이면 생략}
- 실행: (미기록 — 체결 반영 시 갱신: {체결일} / 체결가 {} / 실제 비중 {}%)
- 결과: (미기록 — 복기 시 갱신)
```

기존 항목은 수정하지 않는다(append only). 갱신 가능한 줄은 둘뿐이다 — `실행:` 줄(상태 갱신 모드의 체결 반영 시)과 `결과:` 줄(파이프라인 시작 시 오케스트레이터의 중간점검 `[중간점검 ...]` 또는 복기(trade-reflection)의 확정 `[복기 ...]` 시). **doctor 라벨 정정 규칙:** Phase 7 doctor의 라벨 모순 FAIL로 결정문을 정정하면 정정 항목을 새로 append하고, 원 항목의 `결과:` 줄에 `[대체됨 → {날짜} 라벨 정정]`을 기입한다 — pending 결산·복기·캘리브레이션은 정정 항목만 센다(이중 계상 방지, IPO 부분 재실행 재진입 규칙과 동형). **REVISE 중간 판정 규칙:** REVISE 판정도 저널에 기록하되(감사 추적), append 직후 `결과:` 줄을 `(미기록)`이 아니라 `[REVISE — 최종 확정은 후속 항목에 기록]`으로 즉시 채운다. REVISE·최종 판정 쌍은 1결정이다 — pending 결산·복기·캘리브레이션 집계는 최종 판정 항목만 센다(이중 계상 방지). **포트폴리오 점검 항목 규칙(Phase Q):** PM 모드 3의 점검 항목은 헤더 `## {YYYY-MM-DD} 포트폴리오 점검 — {판정 요지}`(단일 티커 없음)로 쓰고 표준 템플릿의 `실행:`·`결과:` 줄을 **쓰지 않는다** — 매매 결정이 아니므로 pending 결산·복기·캘리브레이션 대상이 아니다(Phase 1 결산·Phase M sweep도 이 항목을 제외한다). 확신도를 표준 앵커로 통일하는 이유: 건마다 정의가 다르면 캘리브레이션 대장(`decisions/calibration.md`)에서 적중률을 집계·비교할 수 없다.
