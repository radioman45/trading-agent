# Filings Lens Toolkit — 공시 해부 4렌즈 공통 방법론

4렌즈는 같은 공시 위에서 **서로 다른 각도**로 같은 회사를 본다. 이 스킬은 4렌즈가 공유하는 보고서 골격·품질 기준과, 렌즈별 세부 방법론(references/)을 담는다.

## 영역 분리 — 중복은 종합을 흐린다

| 렌즈 | 다루는 것 | 다루지 않는 것 |
|------|----------|--------------|
| financial | 재무 3표 숫자, 수익성·현금창출·건전성 추세·구성·정합, 회계 레드플래그 | 경영진 언어, 부문 분해, 위험요인 원문 |
| mdna | 경영진의 실적 설명, 강조/회피, 가이던스·전망·자본배분, 톤 | 숫자 재계산, 부문 구성, 주석 원문 |
| segment | 부문·제품·지역 매출/이익 구성·성장기여·믹스·집중도 | 전사 비율, 경영진 톤, 위험요인 |
| risk | 위험요인, 우발·소송·약정, 특수관계자·부외, 회계단서·후속사건 | 본문 비율, 부문 구성, 경영진 톤 |

겹치면 "숫자 자체는 financial, 숫자에 대한 경영진 언어는 mdna, 숫자의 부문 분해는 segment, 숫자 뒤의 위험·단서는 risk"로 가른다. 인접 항목은 "X 렌즈 참조"로 가리킨다.

## 읽기 순서

1. 이 SKILL.md (공통 규칙)
2. `references/{자기영역}.md` — financial.md / mdna.md / segment.md / risk.md 중 **자기 것 하나만**
3. `_workspace_filings/00_filings_facts.json` (SSOT) — 수치는 SSOT 우선, 세부 라인만 제공 공시 원문에서 직접 확인

## 데이터 경계 [HARD] — 제공 파일만

- 모든 수치·사실은 SSOT와 **제공된 공시 원문**에서만 가져온다. 외부 웹·컨센서스·기억·다른 기간 공시를 끌어오지 않는다.
- 비교는 공시가 스스로 싣는 비교기간(전분기·전년동기·전기말)으로 한다.
- 공시에 없는 것은 추측하지 말고 "공시 미수록"으로 표기한다.
- SSOT 값이 제공 원문과 다르면 보고서 상단에 `⛔SSOT 오류` 플래그(필드/SSOT 값/원문 값/공시 내 위치)를 남기고 원문 값으로 진행한다.

## 공통 보고서 골격 (필수)

모든 렌즈 보고서(`_workspace_filings/01_{영역}_report.md`)는 이 구조를 따른다:

```markdown
# {영역} 해부 보고서 — {회사명} ({공시유형} {회계기간})
- 분석 시점: {YYYY-MM-DD} | 렌즈: {agent-name} | 공시: {source_file}
- (SSOT 오류 발견 시에만) ⛔SSOT 오류: {field} / SSOT 값 {X} / 원문 값 {Y} / 위치 {Z}

## 1. 핵심 요약 (5줄 이내)
{이 렌즈로 본 이번 공시의 상태 진단 — 방향 제시가 아니라 사실 진단}

## 2. 해부 본문
{영역별 references/ 방법론의 필수 섹션}

## 3. 긍정 신호 vs 우려 신호  (※ risk 렌즈는 "실질 위험 vs 완화 요인")
| 긍정 신호 | 근거(공시 내 위치) |
|----------|-------------------|
| 우려 신호 | 근거(공시 내 위치) |

## 4. 데이터 한계
{공시 미수록 항목, 기간·단위 제약 — 없으면 "특이사항 없음"}
```

## 공통 품질 기준

- **중립 의무.** 렌즈는 옹호자도 비관론자도 아니다. §3에서 긍정·우려를 **양쪽 모두** 채운다 — 한쪽이 비면 "이 영역에서 해당 방향 신호 없음"을 명시한다. 종합가가 여기서 균형을 가져가므로 편향된 보고서는 종합을 오염시킨다.
- **사실/해석 분리.** 수치·사실엔 공시 내 위치(출처), 추정·해석엔 "[해석]" 태그.
- **방향 결론 금지.** "매수/매도 추천"은 이 하네스의 목적(이해)을 벗어난 월권이다. 매매 판단은 트레이딩 하네스의 몫이다.
- **추세·구성·정합.** 단일 시점 수준보다 비교기간 대비 변화, 구성비, 항등식 정합을 본다.
- **분량.** 본문 기준 700~1300단어. 짧으면 빈약하고, 길면 종합가의 컨텍스트를 낭비한다.

## 렌즈별 방법론 (필요한 것만 로드)

- `references/financial.md` — 재무 3표 해부, 추세·구성·정합 3축, 회계 레드플래그 체크리스트
- `references/mdna.md` — 경영진 서술 vs 사실 대조, 강조·침묵·완곡어법, 가이던스·자본배분 추출, 톤 신호
- `references/segment.md` — 부문 구성·성장기여 분해·믹스·집중도 분석법
- `references/risk.md` — 보일러플레이트 vs 실질위험 선별, 주석-본문 연결, 회계단서·후속사건 점검
