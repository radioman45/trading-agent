# Analyst Toolkit — 분석가 팀 공통 방법론

분석가 4명은 같은 스냅샷 위에서 **서로 다른 렌즈**로 같은 종목을 본다. 이 스킬은 4명이 공유하는 보고서 골격·품질 기준과, 분석가별 세부 방법론(references/)을 담는다.

## 영역 분리 — 중복은 토론을 죽인다

| 분석가 | 다루는 것 | 다루지 않는 것 |
|--------|----------|--------------|
| fundamental | 재무제표, 수익성·성장성 추이, 밸류에이션, 재무 건전성 | 차트, 뉴스 이벤트, 여론 |
| technical | 가격·거래량, 추세·모멘텀·변동성, 지지/저항 레벨 | 재무, 뉴스 원인 해석, 여론 |
| news | 사건·공시·실적 이벤트, 예정 캘린더, 거시 환경 | 재무 심층, 차트, 여론 반응 |
| sentiment | 투자자 심리, 커뮤니티 여론, 수급·공매도, 컨센서스 쏠림 | 사실 자체(사건·재무·차트) |

겹침이 생기면 "사실은 news, 사실에 대한 반응은 sentiment, 사실의 재무 영향은 fundamental, 가격 반영은 technical"로 가른다.

## 읽기 순서

1. 이 SKILL.md (공통 규칙)
2. `references/{자기영역}.md` — fundamental.md / technical.md / news.md / sentiment.md 중 **자기 것 하나만**
3. `.claude/skills/market-snapshot/references/data-sources.md`의 §0 SSOT 규칙 — 스냅샷 값 우선, 추측 금지, 출처·시점 인용

## 공통 보고서 골격 (필수)

모든 분석가 보고서(`_workspace/01_{영역}_report.md`)는 이 구조를 따른다:

```markdown
# {영역} 분석 보고서 — {회사명} ({티커})
- 분석 시점: {YYYY-MM-DD} | 분석가: {agent-name}
- (스냅샷 오류 발견 시에만) ⛔스냅샷 오류: {field} / 스냅샷 값 {X} / 올바른 값 {Y} / 출처 {Z}

## 1. 핵심 요약 (5줄 이내)
{이 영역에서 본 종목의 현재 상태 — 방향 제시가 아니라 상태 진단}

## 2. 분석 본문
{영역별 references/ 방법론의 필수 섹션}

## 3. 강세 신호 vs 약세 신호
| 강세 신호 | 근거(출처·시점) |
|----------|----------------|
| 약세 신호 | 근거(출처·시점) |

## 4. 데이터 한계
{미확보 항목, 접근 차단, 시점 제약 — 없으면 "특이사항 없음"}
```

## 공통 품질 기준

- **중립 의무.** 분석가는 Bull도 Bear도 아니다. §3에서 강세·약세 신호를 **양쪽 모두** 채운다 — 한쪽이 비어 있으면 "이 영역에서 해당 방향 신호 없음"을 명시한다. 리서처들이 여기서 논거를 가져가므로, 편향된 보고서는 토론을 오염시킨다.
- **사실/해석 분리.** 수치·사실엔 출처·시점, 추정·해석엔 "[추정]"/"[해석]" 태그.
- **방향 결론 금지.** "매수 추천" 같은 결론은 분석가의 월권이다. 방향은 리서치 토론과 매니저가 정한다.
- **분량.** 본문 기준 800~1500단어. 짧으면 빈약하고, 길면 하류 에이전트의 컨텍스트를 낭비한다.

## 분석가별 방법론 (필요한 것만 로드)

- `references/fundamental.md` — 재무 추이 3기간, 밸류에이션 비교축, 레드 플래그 체크리스트
- `references/technical.md` — 00_indicators.json 해석 규칙, 추세/모멘텀/변동성 3축 판정, 가격 레벨 도출
- `references/news.md` — 타임라인·캘린더 작성법, 촉매/리스크 분류, 거시 연결 규칙
- `references/sentiment.md` — 정량 심리 지표 우선순위, 쏠림 판정 스케일, 컨트래리언 해석
