# Fundamental — 기본적 분석 방법론

## 본문 필수 섹션 (§2 분석 본문)

### 2-1. 수익성·성장성 추이 (최소 3개 기간)

스냅샷 `financials`를 기준점으로, 직전 2~3개 기간(분기 또는 연간)을 직접 수집해 추이를 만든다:

- 매출 / 영업이익 / 순이익 — 절대값과 성장률(YoY, 가능하면 QoQ)
- 마진 추이 — 영업이익률·순이익률의 방향 (개선/유지/악화)
- 추세가 수준보다 중요하다: "마진 30%"보다 "25%→30% 개선 중"이 판단을 가른다

### 2-2. 재무 건전성

- 부채 구조: 총부채 vs 현금(스냅샷), 만기 집중 여부, 이자보상배율 [추정 가능]
- 현금흐름: 영업현금흐름과 순이익의 괴리(이익의 질), FCF 방향
- 희석 요인: 증자·전환사채·스톡옵션(SBC) 규모

### 2-3. 밸류에이션

- 스냅샷의 P/E·P/S·EV/EBITDA를 다음 두 축과 비교한다:
  ① **자기 역사** — 과거 3~5년 밴드 대비 현재 위치
  ② **동종 업계** — 핵심 경쟁사 2~3개 대비 프리미엄/디스카운트와 그 정당성
- 멀티플이 정당화되려면 무엇이 사실이어야 하는지(내재 기대치)를 한 줄로 적는다

### 2-4. 사업 구조 (간결히)

- 매출 구성(세그먼트·지역)과 핵심 수익원의 집중도
- 경쟁 우위의 원천과 그것이 약해지는 조건

## 레드 플래그 체크리스트 (의무)

발견 여부와 무관하게 모두 점검하고 결과를 §2에 명시한다:

- [ ] 매출채권이 매출보다 빠르게 증가 (채널 스터핑 신호)
- [ ] 재고 누적 (수요 둔화 신호)
- [ ] 영업현금흐름 ≪ 순이익 (이익의 질 문제)
- [ ] 부채 만기 집중 / 차입 급증
- [ ] 대규모 희석 (증자·CB·SBC)
- [ ] 감사의견·내부회계 특이사항, 빈번한 회계기준 변경

"전 항목 점검, 특이 없음"도 유효한 결과다 — 점검하지 않은 것과 다르다.

## 크립토 자산 특칙 (스냅샷 `market: CRYPTO`)

재무제표 3표와 위 레드 플래그 체크리스트는 발행 주체(법인)가 없어 **적용하지 않는다** — "재무 데이터 없음"을 결함처럼 나열하지 말고, 아래 4축 의무 점검으로 **대체**한다(각 항목 웹 출처 명시):

- [ ] **공급 구조:** 발행 일정·상한, 반감기, 소각(burn), 스테이킹 비율 — 희석 점검의 대응물
- [ ] **온체인 활동:** 활성 주소, 온체인 거래량, 해시레이트/검증자 수 — "사업이 돌아가는가"의 대응물
- [ ] **수급:** 현물 ETF 순유입/유출, 거래소 잔고 추이, 기관·재단 보유 — 현금창출력 점검의 대응물
- [ ] **규제·제도:** 증권성 판단, 과세·회계 기준 변화, 주요국 정책 이벤트

밸류에이션은 PER류 대신 실현시총(Realized Cap)·MVRV·NVT 등 크립토 지표로 앵커한다. 데이터 소스: CoinGecko/CoinMarketCap(유통량·시총), Glassnode·CryptoQuant류(온체인), Farside류(ETF 플로우) — 차단 시 `insane-search`.
