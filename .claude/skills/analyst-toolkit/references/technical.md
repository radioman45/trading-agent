# Technical — 기술적 분석 방법론

## 입력: 00_indicators.json 읽는 법

market-data-engineer가 만든 `_workspace/00_indicators.json`이 SSOT다. **지표를 다시 계산하지 않는다.** 주요 필드:

- `sma.sma20/50/200` + `price_vs_sma_pct` — 이동평균 대비 위치(%)
- `vwma20` — 거래량 가중 이동평균. SMA20과 대조해 추세에 거래량이 실렸는지 본다(가격 > VWMA20 > SMA20이면 매수 거래량 우위, 역순이면 매도 우위)
- `macd.line/signal/histogram/cross` — cross는 직전 영업일 대비 골든/데드 교차 발생 여부
- `rsi14` — Wilder 방식
- `bollinger.position_0to1` — 밴드 내 위치 (0=하단, 1=상단)
- `atr14`, `atr14_pct_of_price` — 손절 폭 설계의 기준
- `returns_pct.1w/1m/3m/6m/1y`, `max_drawdown_pct`, `pct_from_52w_high`
- `volume.last_vs_avg20` — 거래량 확대/위축

## 본문 필수 섹션 (§2 분석 본문)

### 2-1. 추세 판정 (3축 종합)

| 축 | 판정 기준 | 결과 |
|----|----------|------|
| 추세 | 가격 vs SMA50/200, SMA 배열(정배열/역배열) | 상승/하락/횡보 |
| 모멘텀 | MACD 히스토그램 방향, RSI 레벨·방향, 단기 수익률 | 강화/유지/약화 |
| 변동성 | ATR% 추이, 볼린저 폭 | 확대/수축 |

지표 간 **상충을 명시한다** (예: "추세는 상승이나 모멘텀 약화 — 조정 가능성"). 단기(수일~수주)와 중기(수개월) 판정이 다르면 각각 적는다.

### 2-2. 핵심 가격 레벨 (표, 의무)

| 레벨 | 가격 | 근거 |
|------|------|------|
| 저항 1·2 | | 52주 고가, 직전 고점, 볼린저 상단, 라운드넘버 |
| 지지 1·2 | | SMA50/200, 직전 저점, 볼린저 하단 |
| 추세 무효화선 | | 이 가격을 종가로 이탈하면 추세 판정이 뒤집히는 선 |

레벨 근거는 OHLCV(CSV)에서 실제 확인 가능한 것만 쓴다. 트레이더가 이 표로 진입·손절을 설계한다.

### 2-3. 변동성·손절 설계 정보

- ATR14 절대값과 가격 대비 % — "손절 폭은 최소 1.5~2 ATR가 일반적" 같은 설계 참고치 제시
- 최근 갭 발생 빈도(CSV에서 확인) — 갭 리스크가 크면 명시

### 2-4. 시장구조 변동성 레짐 (옵션 감마, 대형주·지수 한정)

대형주·지수 분석 시 `references/market-structure.md`를 읽고 보완한다 — 차트의 변동성을 옵션 구조가 설명할 때가 있다:
- 현재가가 **감마 플립** 위(롱 감마=완충/박스권)인지 아래(숏 감마=추세/갭 가속)인지로 변동성 레짐 태깅.
- 행사가별 감마 집중 지점을 **동적 지지/저항 후보**로(숏 감마 클러스터 = 가속 구간).
- 가장 가까운 OPEX(3째주 금) 민감도. 플립은 매일 이동하는 추정치임을 명기.
- 옵션 유동성이 얕은 종목엔 적용하지 않는다.

### 2-5. 시나리오별 신호

- 강세 확인 신호: "X 돌파 + 거래량 동반이면 추세 지속"
- 약세 확인 신호: "Y 이탈이면 조정 전환"

## 주의

- 거래량 없는 돌파, 표본 부족 패턴(데이터 200일 미만의 SMA200 등)은 신뢰도 낮음을 명시한다.
- 차트가 말하지 않는 것(뉴스 원인, 펀더멘털)을 추측하지 않는다 — 동료 분석가의 영역이다.
