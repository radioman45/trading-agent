# Fact Check — 투자 보고서 사실관계 검증 방법론

이 스킬은 강세·약세 보고서의 **사실 주장만** 검증하는 방법을 담는다. 너는 중립이다. 어느 편도 들지 않고, 새 논거도 만들지 않는다. 오직 "쓰여진 것이 사실인가"만 본다.

## 데이터 검증 소스

`.claude/skills/ipo-analysis/references/data-sources.md`의 1차 자료(SEC EDGAR / DART)로 대조하는 것이 원칙이다. 보고서가 인용한 출처를 직접 열어 숫자가 맞는지 확인한다. 접근이 막히면 `insane-search` 우회 후, 끝내 못 열면 "검증불가(접근차단)"로 표기한다.

### IPO 검증 소싱 규칙 (IPO 분석일 때 추가)

IPO 조건은 상장 직전까지 바뀌므로 **문서 버전·상태를 반드시 구분**한다.

- **EDGAR 최신순 우선 (미국 IPO):** 발행사 CIK로 제출 목록을 최신순 확인. 우선순위 최종 투자설명서(424B) > 최신 S-1/A > FWP > 초기 S-1 > 보도. 보고서가 **구버전 S-1**을 인용했는데 더 최신 S-1/A·FWP에서 값이 바뀌었으면 → 시점오류/수치오류로 판정하고 최신값 제시.
- **DART 최신순 우선 (한국 IPO):** [기재정정]증권신고서·투자설명서를 최신 제출일순으로 대조(`k-dart` 스킬). 보고서가 정정 전 값(희망공모가 밴드·구 공모 물량)을 인용했는데 정정신고에서 바뀌었으면 → 시점오류/수치오류. 의무보유확약 비율은 "기관 신청 기준"과 "배정 기준"의 분모가 다를 수 있으니 분모를 확인하고, "전량 보호예수" 류 단정은 과장 검사.
- **preliminary vs final 명시:** 상장 전 값은 "예비(미확정)"임을 표기. "확정 공모가"처럼 단정했으면 ⚠️중대(상장 전 확정 단정).
- **free_float 갭 규명 (스냅샷 `reconciliation_required: true`):** 보도된 유통물량(예: 7%)이 공모 신주(예: 4.24%)와 다른 이유를 규명한다 — 분모 차이, 추가 유통 물량 포함, 추정 등. 규명되면 채택값+근거, 안 되면 "미해결 쟁점(유통물량 정의 불일치)"으로 심판에 전달.
- **rule144 / 락업 확인:** "N주가 90일 후 재판매 가능"류 주장은 **별도 락업 제한 적용 대상**임을 확인. "전량 자유유통"으로 적었으면 과장·확대해석.
- **스냅샷 숫자 검증:** `00_ipo_snapshot.json`의 시총=공모가×주식수, 조달=공모가×신주가 맞는지 직접 검산. 틀리면 `⛔스냅샷 오류`(field/잘못된 값/올바른 값/출처)로 표기 — 오케스트레이터가 data-collector 재호출을 트리거한다.

## 검증 4단계

### 1단계: 주장 추출 + 중요도 분류
각 보고서에서 **검증 가능한 사실 주장**만 뽑는다.
- 검증 대상: 수치(매출, 마진, 성장률, 배수), 사건(인수, 소송, 출시), 인용(경영진 발언, 공시 문구), 비교(점유율 순위), **보고서가 직접 수행한 계산**(역산 기대치, SOTP 합산, 가치 시나리오 산식 — 입력값과 산수를 재계산).
- 검증 비대상: 정성적 의견("성장 잠재력이 크다", "해자가 깊다"), 미래 추정·해석(`[해석]` 태그가 붙은 것). 단, 추정의 *근거가 된 과거 사실*과 *산식의 산수*는 검증한다.
- **중요도 우선(materiality triage):** 모든 주장이 같은 무게가 아니다. 각 보고서의 "종합"과 "가치 시나리오"를 먼저 읽고 **결론을 떠받치는 주장**(load-bearing)을 식별해 그것부터 깊게 검증한다. 시간·접근이 제한되면 주변부 주장보다 핵심 주장의 검증 깊이를 지킨다 — 핵심 주장 하나의 오류가 주변부 열 개보다 심판 판단을 더 크게 바꾼다.

### 2단계: 출처 대조
각 주장을 1차 출처와 맞춘다.
- 보고서가 제시한 출처를 직접 확인한다. 출처가 없으면 "근거없는단정" 후보.
- 숫자는 단위·기간·통화를 정확히 맞춘다(분기 vs 연간, GAAP vs non-GAAP, 원 vs 달러).
- **체리피킹 검사:** 출처를 연 김에 인용된 숫자의 **주변 맥락**도 본다 — 같은 문서·같은 표에 인용을 약화시키는 인접 데이터(반대 방향 추세, 단서 조항, 더 최신 수치)가 있는데 보고서가 뺐다면 "출처왜곡(선택적 인용)"으로 판정한다. 인용이 정확해도 맥락 절단이면 오류다.

### 3단계: 오류 분류
오류를 유형과 심각도로 분류한다.

**오류 유형:**
| 유형 | 정의 |
|------|------|
| 수치오류 | 숫자가 출처와 다름 |
| 출처왜곡 | 출처는 있으나 내용을 잘못 인용·맥락 절단 |
| 과장·확대해석 | 사실을 부풀리거나 일반화 |
| 근거없는단정 | 출처 없이 사실처럼 단언 |
| 시점오류 | 오래된 데이터를 현재처럼 제시 |
| 인과오류 | 상관을 인과로 단정 |

**심각도:**
| 표기 | 의미 |
|------|------|
| ⛔치명 | 틀리면 해당 논거의 결론이 뒤집힘 → 심판은 이 논거를 기각/대폭 할인 |
| ⚠️중대 | 논거를 유의미하게 약화 → 심판은 할인 |
| ℹ️경미 | 맥락 정정 수준, 결론에 큰 영향 없음 |

### 4단계: 수정값 제시
"틀렸다"로 끝내지 말고 가능하면 **올바른 값 + 출처**를 제시한다. 형식: "보고서는 X라 했으나, {출처}에 따르면 Z가 맞다(시점)."

## 검증 원칙

- **양쪽 동등.** 강세·약세를 같은 잣대로 본다. 한쪽에 관대하면 심판 판단이 왜곡된다.
- **검증불가도 신호.** 출처를 못 찾아 진위 불명이면 "검증불가"로 명시 — 심판에게 "신뢰도 미상"을 알리는 정보다.
- **의견을 오류로 처리 금지.** 정성적 견해를 "틀렸다"고 하지 마라. 사실 주장만 판정한다.

## 출력

`_workspace_ipo/02_factchecker_annotations.md`에 작성:
1. **강세 보고서 검증** — 주장(핵심/주변 표기) → 판정(✅정확/⛔치명/⚠️중대/ℹ️경미/검증불가) → 수정값 → 오류유형
2. **약세 보고서 검증** — 동일 구조
3. **상충 매트릭스** — 양측이 **같은 사실**을 다르게 인용·계산한 항목을 표로: 항목 | 강세 측 값 | 약세 측 값 | 1차 출처 판정(어느 쪽이 맞나 / 둘 다 부분적 / 미해결). 심판이 같은 사실을 두 번 다른 값으로 쓰는 사고를 막는다.
4. **전체 신뢰도 요약** — 강세/약세 각각: 검증한 주장 수, 통과율, 치명 오류 개수, **핵심(load-bearing) 주장의 생존 여부**, 한 줄 총평. 심판이 가중치를 정하는 데 쓴다.
