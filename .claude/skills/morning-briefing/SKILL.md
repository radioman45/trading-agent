# Morning Briefing — 나 맞춤형 국제 금융 속보

**벤치마크:** 국제금융센터(KCIF) 국제금융속보(Flash News, `sources/국제금융속보/` 견본 4부). 그 수준의 핵심은 "수치 + 상충하는 기관 해석 병기"다. **차별점:** 이 브리핑은 레포의 학습 코퍼스(보유 SSOT·저널 트리거·워치리스트)를 읽고 시작하는 **개인화 레이어**가 본체다 — 같은 뉴스라도 "내 포트폴리오에 무엇인가"로 번역한다.

**경계:** 정보 브리핑이다. 매수/매도 판단·비중 권고를 하지 않는다(그건 trading-strategy 풀 파이프라인·Phase Q의 몫). 트리거 "발동" 표시는 판단이 아니라 **사용자가 결정 당시 스스로 만든 규칙의 기계적 대조**다.

## 0. 시점 앵커 (필수 — 할루시네이션 가드)

1. **KST(UTC+9) 기준으로 날짜·요일을 계산한다 — UTC 요일로 분기 금지.** 이 루틴은 UTC 22:00에 돌므로 컨테이너 `date`(UTC)는 **항상 KST 기준 전날**을 보여준다(UTC 22시 = KST 익일 07시). `TZ='Asia/Seoul' date` 또는 UTC+9 수동 가산으로 KST를 구하고, **모든 날짜 스탬프·파일명·요일 분기·메일 제목은 KST 값**을 쓴다. 브리핑 상단에 UTC·KST를 병기해 검산 흔적을 남긴다.
2. **마감된 세션의 수치만 사실로 서술한다.** 미국 화~토 아침(KST)은 전일 뉴욕 마감이 최신 확정. 월요일 아침은 금요일 마감이 최신(주말 미국·한국 휴장 — "지난주 마감" 명시). 크립토는 24/7(당일 UTC 봉 미확정 명시).
3. 모든 수치에 출처를 단다(야후/스크립트/기사명). 확보 실패 값은 지어내지 않고 "미확보"로 표기한 채 발행한다 — **발행 자체를 막지 않는다**(부분 결측 브리핑 > 무브리핑).

## 1. 요일 분기

| 요일(KST 아침) | 모드 | 내용 |
|---------------|------|------|
| 화~토 | **마감 속보** (기본) | 전일 뉴욕 마감 + 밤사이 뉴스 |
| 월 | **주간 프리뷰** | 지난주 요약(간략) + 이번 주 캘린더·내 트리거 예정·아시아 개장 동향 중심 |
| 일 | (루틴 미실행) | — |

**UTC↔KST 대조(오프바이원 방지):** UTC 일요일 22시 = **KST 월요일 아침 → 주간 프리뷰**(미실행 아님!) / UTC 금요일 22시 = KST 토요일 아침 → 마감 속보 / cron이 UTC 토요일 22시(=KST 일요일)를 이미 제외하므로 "일=미실행" 행은 KST 기준 안전장치다.

## 2. 데이터 수집

- **의존성 부트스트랩(수집 전 필수):** `python -c "import yfinance, pandas"` 확인 — 실패 시 `pip install yfinance pandas numpy` 후 재확인. 그래도 실패할 때만 뉴스판 강등을 허용한다(침묵 강등 금지 — 강등 사유를 브리핑 상단에 명시).
- **시장 대시보드:** python 원라이너(yfinance fast_info)로 일괄 — `^GSPC`(S&P500)·`^IXIC`(나스닥)·`^STOXX`(Stoxx600, 실패 시 생략)·`^N225`(닛케이)·`^KS11`(KOSPI)·`^TNX`(미10Y — 소스에 따라 42.5(=4.25%) 또는 4.25로 오므로 **4~6% 범위 sanity check 후 표기**)·`DX-Y.NYB`(달러지수)·`KRW=X`(원달러)·`^VIX`·`BZ=F`(브렌트)·`GC=F`(금)·`BTC-USD`. 실패 티커는 WebSearch로 보완, 그래도 없으면 "미확보".
- **보유 종목 시세:** `decisions/portfolio.json`의 holdings 티커. **한국 티커(6자리 숫자·신형 영숫자 — market:"KR")는 yfinance에 `.KS`(→실패 시 `.KQ`) 접미사를 붙여야 한다**(예: `0167A0.KS`, `088980.KS` — 원시 코드 그대로는 반환 없음). 실패 시 번들 스크립트(`.claude/skills/market-snapshot/scripts/collect_market_data.py --ticker {t} --out-dir /tmp/brief/{t}` — KR 자동 접미사·신형 영숫자 대응 내장)로 폴백. 브리핑엔 fast_info 수준(전일 종가·등락률)이면 충분하다. **KR 지수·보유가 직전 영업일과 동일 값이면 한국 휴장 가능성 — "휴장/스테일 가능" 표기.**
- **뉴스:** WebSearch 3~5회 — ① 거시(Fed·금리·CPI·고용 등 밤사이 핵심) ② 보유 종목별 중대 뉴스(TSLA 등 — 실적·규제·급등락 원인) ③ AI/반도체 섹터(포트폴리오 집중 팩터) ④ 한국 시장·환율. 가능하면 KCIF 최신 속보(kcif.or.kr)도 WebFetch로 참조해 톤을 맞춘다(차단 시 생략).
- **개인화 원천(레포):** `decisions/portfolio.json`(보유·비중·바벨), `decisions/journal.md`(열린 결정의 무효화 조건·게이트 일정·`실행:` 줄), `decisions/watchlist.md`(있으면), 최근 `reports/포트폴리오점검_*/`(있으면 — 재점검 트리거).

## 3. 산출 형식 (`briefings/모닝브리핑_{YYYY-MM-DD}.md`)

```markdown
# 모닝브리핑 — {YYYY-MM-DD}({요일}) 07:00 KST
> 기준: 뉴욕 {날짜} 마감 · 작성 {UTC/KST 시각} · 모드 {마감 속보|주간 프리뷰}

## 📌 주요 뉴스 (3~5줄, 내 포트폴리오 관련도순)
## 📊 시장 대시보드
| 구분 | 지표 | 종가 | 등락 |  (S&P500·나스닥·닛케이·KOSPI / 미10Y·달러지수·원달러 / VIX·브렌트·금·BTC)
## 🔍 금일의 포커스 (1건 심층)
{데이터 → 해석 → 기관 견해 인용(상충 시 병기) → 내 포트폴리오 함의 1줄}
## 💼 내 포트폴리오 레이더
| 종목 | 비중 | 전일 등락 | 밤사이 뉴스/신호 |  (뉴스 없으면 "특이 없음" — 지어내지 않는다)
## 🚨 트리거 보드 (저널 열린 결정 대조 — 판단 아님, 규칙 대조)
| 트리거 | 기준 | 현재 | 상태(발동🔴/근접🟡 ±3%/정상🟢) |
## 🌍 글로벌 동향 (3~5건, 각 1~2문단)
## 🗓️ 오늘·이번 주 캘린더 (내 트리거와 겹치면 🔔)
## 💬 해외시각 (IB 뷰 1~3건 — 출처 명시)
```

트리거 보드는 **매일 `decisions/journal.md`의 열린 항목(무효화 조건·게이트)에서 재구성한다** — 아래 목록은 2026-07-05 기준 스냅샷 예시로, 검증 기대치로만 쓰고 그대로 복사하지 않는다(저널이 항상 우선): AGNC 7/14 CPI·7/27 Q2 BV 게이트 / NVDA 8/26 실적·$214 돌파·$199 이탈 / MKIF 8월 IR·10,000 종가 / SOL(0167A0) 22,130 전량·24,471 부분 / 레짐 무효화(Fed 10월 인상 확정·한국 CPI 3.5%↑·원달러 1,600↑) / Phase Q 재점검(TSLA 30%/RC45%·AI RC 95%·SOL 15%).

## 4. 전달

1. **레포 아카이브:** `briefings/모닝브리핑_{KST날짜}.md` 저장 → `git add briefings/ && git commit -m "브리핑: {KST날짜}" && git push`. push 거부 시 브랜치 `briefing/{날짜}`로 push하고 그 사실을 메일에 명시.
2. **Gmail 초안:** 수신 `yh.lee223@gmail.com`, 제목 `[모닝브리핑 {KST MM/DD}] {주요 뉴스 1줄 요약}`, 본문은 브리핑 전문(마크다운 그대로). 초안 생성이지 발송이 아니다. **중복 가드:** 같은 날짜 제목의 초안이 이미 있으면(재실행 케이스) 새로 만들지 않는다. Gmail 도구 부재/실패 시 커밋만으로 완료 처리하고 실패를 커밋 메시지에 남긴다.
3. **최후 폴백:** git push와 Gmail이 **둘 다** 실패하면 브리핑 전문을 세션 최종 메시지로 출력한다(클라우드 세션 결과에 보존 — 브리핑 증발 금지).

## 5. 품질 규율

- **관련도 우선순위:** 이 사용자는 AI 팩터 집중(TSLA·SOL반도체·NVDA·PLTR)·금리 인컴(AGNC·MKIF) 보유 — AI 캐펙스·금리·환율 뉴스가 항상 상위. 단 우선순위는 portfolio.json을 읽어 동적으로 정한다(보유가 바뀌면 따라간다).
- **기관 견해는 인용으로:** "UBS는 ~라고 해석(출처)" 형식. 견해가 상충하면 병기한다 — 한쪽만 고르지 않는다.
- **길이:** 본문 기준 A4 1.5~2장(KCIF 1건 분량). 뉴스가 없는 날은 짧게 — 채우기 금지.
- **매매 언어 금지:** "사라/팔아라/비중 늘려라" 금지. 트리거 발동 시에도 "결정 당시 규칙상 점검 시점 — 포지션 점검 또는 풀 파이프라인 실행을 고려"까지만.
