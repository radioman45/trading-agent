# Portfolio Risk — 포트폴리오 영향 분석

**보유 비중 ≠ 리스크 비중.** 종목 단위 게이트(손절·비중 상한)만으로는 "타이트한 손절 덕에 허용된 35% 단일 종목"이 포트폴리오 전체를 하나의 거시 변수(예: 한국 장기금리)에 노출시키는 것을 막지 못한다. 이 스킬은 신규 거래를 **사용자의 실제 보유 북 위에 얹었을 때** 무엇이 달라지는지를 측정한다. 측정이지 판정이 아니다 — 최종 게이트는 portfolio-manager의 몫이다.

## 입력: 보유 현황 SSOT (`decisions/portfolio.json`)

사용자 보유 현황의 단일 진실 소스. 스키마:

```json
{
  "as_of": "YYYY-MM-DD",
  "base_currency": "KRW",
  "cash_pct": 30.0,
  "holdings": [
    {
      "ticker": "MKIF", "name": "맥쿼리인프라", "market": "KR",
      "weight_pct": 15.0, "avg_price": 12345, "currency": "KRW",
      "barbell_leg": "방어(인컴)",
      "macro_drivers": ["한국 장기금리", "국내 인프라 규제"]
    }
  ]
}
```

- `weight_pct`: 계좌 총액 대비 %. 합계 + cash_pct ≈ 100 검산.
- `barbell_leg`: 공격(AI/성장) | 방어(원자재/인컴/현금흐름) | 중립. **크립토 보유는 공격으로 태깅**(위기 상관 수렴 — risk-gate 바벨 규칙과 동일). 크립토의 `macro_drivers` 예: 글로벌 유동성, 달러, 현물 ETF 수급.
- `macro_drivers`: 이 자산의 손익을 지배하는 공통 거시 변수 1~3개(금리·달러·AI 캐펙스 사이클·유가·반도체 사이클 등). **상관 근사의 핵심 입력이다.**
- **갱신 시점:** 사용자가 "보유 등록/갱신"을 요청할 때, 그리고 "체결 반영" 시 오케스트레이터가 갱신한다. 파일이 없으면 분석을 강등한다(아래 "보유 미등록" 참조).

## 포트폴리오 모드 판정 (분석 시작 시 가장 먼저 — 라벨-실체 일치)

`portfolio.json` 상태로 이번 분석의 모드를 판정하고 산출물 최상단에 표기한다. PM 게이트가 이 라벨을 요약 박스에 그대로 옮긴다:

| 모드 | 조건 | 의미 |
|------|------|------|
| `PORTFOLIO_AWARE` | holdings ≥ 1건 + as_of가 90일 이내 | 실제 보유 북 기준의 집중·상관·꼬리 측정 |
| `PORTFOLIO_STALE` | holdings ≥ 1건이나 as_of가 90일 초과 또는 미기재 | 측정은 수행하되 "보유 스냅샷이 낡음 — 갱신 권고"를 관찰에 명시 |
| `SINGLE_TRADE_ONLY` | holdings 비어 있음(강등 모드) | 신규 캠페인 단독 꼬리 검산만 — 포트폴리오 판단 아님 |

**표현 금지 규칙:** `SINGLE_TRADE_ONLY`에서는 산출물·결정문 어디에도 "포트폴리오를 반영/고려했다", "portfolio-aware"류 표현을 쓰지 않는다. 실제로 잰 것은 단일 캠페인의 꼬리뿐인데 라벨이 포트폴리오 판단처럼 읽히면, 사용자가 하지 않은 검증을 받았다고 오독한다.

## 계산 절차 (산술 자가검산 의무)

모든 수치에 `[관측|출처]` 또는 `[추정|근거]` 태그. 계산은 저장 전 재검산한다.

### 1. 종목 변동성
- 신규 종목: `00_ohlcv_daily.csv`의 일수익률 표준편차 × √252 = 연환산 변동성.
- 기보유 종목: 가능하면 수집 스크립트로 산출, 불가하면 자산군 대표치로 `[추정]`(예: 대형주 20~25%, 인컴/인프라 10~15%, 원자재 25~35%).

### 2. 상관 근사 (데이터 없을 때의 규율)
쌍별 상관을 계산할 데이터가 없으면 **macro_drivers 겹침 기준 3단계 근사**를 쓰고 반드시 `[추정]` 태그:
| 관계 | 근사 상관 ρ |
|------|------------|
| 같은 거시 변수 + 같은 시장/섹터 | 0.8 |
| 거시 변수 1개 공유 또는 같은 바벨 다리 | 0.5 |
| 공유 변수 없음·다른 다리 | 0.2 |

### 3. 리스크 기여 근사
```
RC_i ∝ w_i × σ_i × ρ_i,p   (비중 × 변동성 × 포트폴리오와의 상관)
```
전 종목 RC를 정규화해 %로 표기(합=100%). **비중 상위와 RC 상위가 다르면 그 자체가 발견이다.**

### 4. 집중 점검 (신규 편입 전 → 후 델타)
- **단일 종목:** 편입 후 최대 종목의 비중과 RC.
- **단일 거시 변수:** 같은 macro_driver를 공유하는 포지션의 비중 합산과 RC 합산 (예: "한국 장기금리" 노출 = MKIF + 리츠 + 장기채 합산).
- **바벨 균형:** 공격/방어/현금 3분할이 `00_macro_regime.md`의 바벨 기울기 권고와 정합하는가.
- **통화:** 원화 기준 외화 노출 합계(환헤지 여부 가정 명시).

### 5. 동시 꼬리 손실 (aggregate heat)
```
동시 꼬리 손실(%) = Σ ( w_i × 꼬리 시나리오 손실률_i )    — 같은 거시 충격에 동반 노출된 포지션만 합산
```
- 꼬리 시나리오 손실률: 해당 결정의 `06_final_decision.md` 잔여 리스크(하방 이론가) 또는 -2σ 주간 변동 중 큰 쪽.
- **해석 기준:** 단일 거시 충격 시 동반 손실 합이 계좌 10%를 넘으면 🚩 집중 경고(기본값 — 사용자 조정 가능). 신규 캠페인 단독으로는 "비중 × 꼬리 손실률 ≤ 계좌 2%"(trade-planning의 꼬리 검산과 동일 산식)를 확인한다.

## 보유 미등록 시 (강등 모드 = `SINGLE_TRADE_ONLY`)

`decisions/portfolio.json`이 없거나 holdings가 비어 있으면 추측으로 채우지 않는다. 산출물에 다음만 쓴다:
1. "모드: SINGLE_TRADE_ONLY — 보유 미등록, 포트폴리오 영향 분석 불가. 신규 캠페인 단독 꼬리 검산만 수행."
2. 신규 캠페인 꼬리 검산(위 산식)은 수행한다 — 보유 정보 없이도 가능하다.
3. 등록 안내 1줄: "『보유 등록』이라고 요청하면 1회 인터뷰로 portfolio.json을 만든다."

## 한계 (정직하게 명시)

- 3단계 상관 근사는 위기 시 상관 급등(동반 폭락)을 과소평가할 수 있다 — 꼬리 손실 합산이 이를 부분 보완한다.
- 과거 변동성은 사후 추정치다. 백테스트 샤프와 마찬가지로 절대 신뢰 금지(method_portfolio_risk.md의 한계 절 참조).
- 이 분석은 리밸런싱 지시가 아니다. "의도한 리스크 구조와 실제의 일치 여부"를 보여줄 뿐, 행동 결정은 PM 게이트와 사용자의 몫이다.
