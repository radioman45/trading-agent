# Idea Scan — 아이디어 발굴·스크리닝

반응형 시스템의 알파 상한은 사용자의 착상에 묶인다 — 사용자가 가져온 티커를 "약간 더 잘 분석"하는 것으로는 좋은 기회를 먼저 찾을 수 없다. 이 스킬은 거시 레짐 판정을 **구체적 후보 리스트**로 번역해, 파이프라인에 능동적 입구를 만든다. **스캔의 산출물은 매수 추천이 아니라 "풀 분석할 가치가 있는 후보 3~5개 + 근거"다.**

## 절차

### 1. 레짐 앵커 (필수 선행)
`_workspace/00_macro_regime.md`를 읽는다. **없거나 판정 시점이 7일 이상 지났으면 macro-strategist를 먼저 실행**해 갱신한다 — 낡은 레짐 위의 스캔은 낡은 후보를 낳는다. `decisions/lessons.md`의 관련 교훈도 반영한다.

### 2. 레짐 → 테마 번역
레짐 판정을 기계적으로 자산군·섹터·스타일로 번역한다. 번역표(출발점 — 레짐 보고서의 구체 권고가 우선한다):

| 레짐 축 | 기울기 | 후보 방향 |
|---------|--------|----------|
| risk-on + 유동성 완화 | 공격 다리 | AI/성장 테마, 고베타, 소형 성장 |
| risk-off + 긴축 | 방어 다리 | 원자재·인컴·저변동 배당, 현금 비중 |
| 전환점(고점 경계) | 바벨 양끝 | 공격·방어 동시 보유, 중간(어중간한 밸류 성장) 회피 |
| 성장↑물가↑ | 실물 | 원자재·에너지·소재 |
| 성장↓물가↓ | 듀레이션 | 장기채·성장주(할인율 수혜) |

각 테마당 대표 후보 2~4개(개별 종목 + ETF 혼합)를 수집한다. 후보 수집 시 **출처·시점 의무** — 웹 검색·스크리너 결과는 조회 URL과 날짜를 남기고, 기억으로 티커를 나열할 때는 `[기억 — 시세·펀더멘털 미확인]` 태그를 붙인다(할루시네이션 가드 준용).

### 3. 정량 1차 필터 (후보당 3분 체크)
후보마다 아래 3축을 ○/△/× 로 빠르게 채점한다. 정밀 분석은 풀 파이프라인의 몫 — 여기서는 명백한 탈락만 거른다:
- **밸류:** 섹터 대비 명백한 고평가 아님 (P/E·P/S 상대 비교 1개면 충분)
- **모멘텀/추세:** 명백한 하락 추세 한가운데 아님 (52주 위치·최근 3개월 방향)
- **퀄리티:** 적자 지속·희석 남발·거버넌스 적신호 없음

### 4. 촉매 캘린더 (향후 4~8주)
후보별로 시간축 위의 촉매를 적는다: 실적 발표, FOMC/금통위, 지수 리밸런싱, 락업 해제, 제품/규제 이벤트. **촉매 없는 후보는 우선순위를 낮춘다** — "언젠가 오를 것"은 자본 회전율의 적이다.

### 5. 역발상 점검 (쏠림 테마 경계)
`.claude/skills/contrarian-check/SKILL.md`를 적용한다. 스캔이 뽑은 테마가 이미 미디어·커뮤니티에서 과열이면(모두가 아는 테마) 후반부 진입 리스크를 명시하고, 반대로 **비관 쏠림 속 생존 후보**(컨센서스가 외면하지만 1차 필터를 통과)에 가점을 준다.

### 6. 우선순위 스코어링 → 추천
각 후보를 `레짐 정합(0~3) + 1차 필터(0~3) + 촉매 근접(0~2) + 역발상 가점(0~2)`으로 스코어링하고 상위 3~5개를 **풀 파이프라인 분석 추천**으로 올린다. 나머지는 워치리스트에 조건과 함께 보관한다.

## 워치리스트 관리 (`decisions/watchlist.md`)

스캔 종료 시 갱신한다. 항목 포맷:

```markdown
| 종목(티커) | 테마/다리 | 상태 | 진입 검토 조건 | 다음 촉매 | 등재일 | 재검토일 |
```

- **상태:** 후보(스캔 통과) / 분석완료(풀 파이프라인 거침 — journal 참조) / 보류(조건 미충족) / 기각(사유 명기)
- **기각도 기록한다** — "왜 안 샀는가"는 복기 때 기회비용 채점의 원료다.
- 재검토일이 지난 항목은 다음 스캔에서 최우선 재점검한다.

## 규율

- **스캔 ≠ 추천.** 산출물 말미에 "이 리스트는 분석 후보이며 매수 추천이 아니다. 진입 판단은 풀 파이프라인(분석→토론→게이트)을 거쳐야 한다"를 명시한다.
- **시점·출처 가드 준용.** 시세·지표를 인용하면 조회 시점을 남기고, 마감 안 된 세션 수치를 단정하지 않는다.
- **레짐 없이 스캔하지 않는다.** 레짐 미확보 시 스캔을 중단하고 macro-strategist 선행을 요청한다 — 테마 없는 스크리닝은 지표 낚시(factor fishing)다.
