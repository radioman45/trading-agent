# 보고서 스키마 — 강세/약세/최종 보고서 공통 구조

Bull과 Bear의 보고서를 동일 구조로 맞추면 팩트체커가 교차 검증하기 쉽고, 심판이 나란히 비교하기 쉽다. 아래 구조를 따른다.

## 공통 헤더 (Bull·Bear 모두)

```markdown
# [강세론|약세론] — {회사명} ({티커}, {시장})
- 분석 시점: {YYYY-MM-DD}
- 현재 주가 / 시가총액: {값} (출처)
- 입장: 이 보고서는 의도적으로 {강세|약세} 한쪽만 변호한다.
```

## Bull / Bear 본문 구조

각 논거를 아래 형식의 카드로 작성한다. 최소 5개 축.

```markdown
## 논거 {N}: {제목}
- **주장:** {한 문장 핵심 주장}
- **근거:** {수치 + 출처 + 시점}
- **확신도:** 강 | 중 | 약
- **컨센서스 대비:** {시장/공모가가 이미 반영한 것 vs 이 논거의 차별점. 이미 가격에 반영된 사실이면 명시}
- **반증 신호:** {이 논거가 틀렸다면 관찰될 구체적 지표·이벤트 1개 이상}
- **반대측 예상 반박과 재반박:** {선택}
```

> **컨센서스 대비**가 없는 논거는 "맞지만 이미 가격에 반영된" 죽은 논거일 수 있다 — 심판이 이 필드로 변별력을 가중한다. **반증 신호**는 심판의 반증 조건과 진입 전략의 가설훼손 체크포인트로 직결된다.

본문 뒤 필수 섹션 (4개):

```markdown
## 가치 시나리오 (정량)
이 입장이 실현될 때의 가치 경로. 페이오프 크기는 산식과 함께, 발생 가능성 평가는 [해석] 태그로 분리(최종 확률 배분은 심판 소관).
| 시나리오 | 핵심 가정 | 주당 가치 / 페이오프 (산식) | 실현 조건·시점 |
|---------|----------|---------------------------|---------------|

## 프리모템 (자기반증)
{N년 후 이 입장이 완전히 틀린 것으로 판명됐다고 가정하고, 가장 그럴듯한 패인 1~2개를 스스로 적는다. 상대측 논거 복사가 아니라 "내 논거의 가장 약한 고리"여야 한다.}

## 종합
{이 입장의 핵심 3줄 요약}

## 검증 요청 항목
팩트체커가 우선 확인할 핵심 수치·주장 (각 항목에 내가 사용한 출처 명시). **결론을 떠받치는 무게 순**으로 정렬:
1. ...
2. ...
```

## 최종 보고서 구조 (Judge 전용)

`_workspace_ipo/03_judge_verdict.md`는 사용자가 실제로 읽는 산출물이다. 아래 순서를 따른다.

```markdown
# 투자 판단 — {회사명} ({티커})

## 결론
> **{BUY | HOLD | SELL}** · 확신도 {0~100}%
> *(확신도 정의 — 표준 앵커: "12개월 내 벤치마크 대비 초과수익 확률". 다른 정의 필요 시 표준 앵커 값 병기)*
{한 문단 요약: 왜 이 결론인가}

## 검증된 강세 요지
{팩트체크를 통과한 강세 논거만, 강한 것부터}

## 검증된 약세 요지
{팩트체크를 통과한 약세 논거만, 강한 것부터}

## 팩트체크가 바꾼 것
{어떤 논거가 무너졌나 — 치명/중대 오류로 기각·할인된 주장}

## 크럭스 (결정적 쟁점)
{강세·약세를 가르는 단일 핵심 질문 1~2개. "이 질문의 답이 결론을 정한다"는 쟁점과, 현재 증거가 어느 쪽을 가리키는지}

## 비대칭 분석 · 기대값 표
{상방(크기×확률) vs 하방(크기×확률). 꼬리 위험 명시. 시나리오별 기대값을 표로 명시 계산:}
| 시나리오 | 확률 | 확률 근거 (기저율 → 사례 조정) | 페이오프 (기준가 대비) | 기여 EV |
|---------|------|------------------------------|----------------------|---------|
{확률 합 = 100% (상호배타·전수). EV 합계와 결론의 정합을 한 줄로 확인 — EV가 음수인데 BUY, 뚜렷이 양수인데 SELL이면 모순}

## 최종 판단 논리
{양측을 사실로 가중해 결론에 이른 추론. 컨센서스(가격에 이미 반영된 것) 대비 이 판단의 차별적 관점이 무엇인지 포함}

## 모니터링 지표 · 반증 조건
{이 판단을 뒤집을 구체적 조건과, 추적할 지표}

## (선택) 투자 성향별 코멘트
{장기/단기, 리스크 허용도별 시사점 — 입력에 사용자 성향이 있을 때}

---
*면책: 본 보고서는 AI 에이전트가 공개 자료를 바탕으로 생성한 분석이며 투자 권유가 아니다. 최종 투자 판단과 책임은 투자자 본인에게 있다.*
```

## IPO 변형 (IPO 분석일 때 — 위 구조를 다음으로 치환)

IPO는 거래가·과거 멀티플이 없으므로 "현재 주가/목표가/손절가/12개월 reward·risk" 필드를 IPO 필드로 바꾼다.

**Bull/Bear 공통 헤더 (IPO):**
```markdown
# [강세론|약세론] — {회사명} ({티커}, IPO)
- 분석 시점: {YYYY-MM-DD} · 상장(거래개시) 예정: {YYYY-MM-DD}
- 예비 공모가 / 상장후 시총: {값} (출처·preliminary)
- 입장: 이 보고서는 의도적으로 {강세|약세} 한쪽만 변호한다.
```
> "현재 주가"가 아니라 "예비 공모가"를 쓴다. 거래 멀티플(PER 등) 대신 SOTP/EV 배분·세그먼트 단위경제성·가치 앵커를 근거로 한다.

**최종 보고서 (Judge, IPO) — "## 결론" 섹션을 다음으로 치환:**
```markdown
## 결론
> **{BUY | HOLD | SELL}** · 확신도 {0~100}%
> *(확신도 정의 — 표준 앵커: "상장 후 12개월 내 벤치마크 대비 초과수익을 낼 주관 확률". 캘리브레이션 채점(벤치마크 알파 부호)과 같은 사건. 보조 정의 필요 시 병기)*

### 공모가 대비 가치 정합
- 예비 공모가: ${X} / 상장후 시총: ${Y}T
- 가치 앵커: 현 실적 기반 바닥 ${a}~${b}/주, 공격적 포워드 ${c}~${d}/주
- 옵션 프리미엄 비중: 공모가의 약 {Z}%가 미실현 옵션({종목의 미검증 신사업·수급·실행력 등을 명시})

### 진입 가치 구간 (단일 목표가 금지)
- ${P1} 이하: 바닥가치 매력 / ${P2} 이상: 옵션에 과지불
```
> IPO에서 "비대칭 분석 · 기대값 표"의 페이오프 기준가는 **예비 공모가**다(거래가가 없으므로). 기저율 prior는 동급 코호트(메가 IPO·고멀티플 신규상장)의 평균 성과를 쓰되, 정량 통계 미확보 시 "정량 기저율 미확보(정성적 외부 관점)"로 정직하게 표기한다.
이어서 "검증된 강세/약세 요지", "팩트체크가 바꾼 것", "비대칭 분석(락업·free float·안정조작·지수 반영)", "최종 판단 논리", "모니터링 지표·가설훼손 조건"을 둔다. **IPO 고유 리스크**(락업 타임라인·free float 갭·안정조작 종료·지수편입 선반영)를 비대칭 하방에 명시한다.

## 면책 조항 규칙

최종 보고서에는 반드시 위와 같은 **투자 권유 아님** 면책 문구를 포함한다. IPO는 상장 전 조건이 최종 투자설명서에서 바뀔 수 있고 한국 투자자의 실제 매수 가능 여부가 증권사별로 다르다는 점도 함께 명시한다. 이는 법적 안전장치이자 사용자에게 "이건 의사결정 보조 도구"임을 상기시킨다.
