# 데이터 소스 가이드 — 시장 자동 감지 + 무료 1차 자료

Bull·Bear·Fact-checker·data-collector가 공통으로 사용하는 데이터 수집 규칙. 분석 대상이 미국 주식인지 한국 주식인지 자동으로 판별하고, 해당 시장의 1차 자료를 우선 사용한다.

## 0. 단일 진실 소스 (SSOT) — 스냅샷 우선 [최우선]

`_workspace_ipo/00_ipo_snapshot.json`이 존재하면, 그 안에 고정된 값은 **단일 진실 소스**다. 모든 에이전트는 다음 항목을 스냅샷 값으로만 사용한다(직접 찾은 다른 값으로 대체 금지):

- 주가(`price`), 시가총액(`market_cap`), 발행주식수(`shares_outstanding`)
- 환율(`fx`), 벤치마크 지수 레벨(`benchmark`)
- 재무 핵심값(`financials`: 매출·영업이익·순이익·부채·현금), 밸류에이션 배수(`valuation`: P/E·P/S·EV/EBITDA)

스냅샷에 **없는** 항목(세그먼트별 매출, 정성 정보, 업종 동향 등)만 직접 수집한다. 스냅샷의 `confidence: "unavailable"` 필드는 "그 값은 미확보"라는 신호이므로 추측으로 메우지 말고 미확보로 다룬다.

**예외 — fact-checker:** fact-checker는 스냅샷 값 자체를 1차 출처와 대조 검증할 권한을 가진다(아래 에이전트별 지침 참조).

## 1. 시장 자동 감지

분석 대상(회사명/티커)을 받으면 먼저 어느 시장인지 판별한다:

| 단서 | 판정 | 예시 |
|------|------|------|
| 알파벳 티커 (1~5자), NYSE/NASDAQ 언급 | **미국** | AAPL, NVDA, TSLA, MSFT |
| 6자리 숫자 코드, KOSPI/KOSDAQ, 한글 회사명 | **한국** | 005930(삼성전자), 035720(카카오) |
| 영문 회사명이지만 한국 기업 | **한국** | Samsung Electronics, Hyundai |
| 판별 모호 | 웹 검색으로 상장 거래소 확인 후 결정 | - |

티커가 모호하면(예: "ARM"은 미국 상장) 웹 검색 한 번으로 상장 거래소를 확정한 뒤 진행한다.

## 2. 미국 주식 데이터 소스

- **SEC EDGAR** (1차, 무료): 10-K(연간), 10-Q(분기), 8-K(수시), DEF 14A(위임장). `database-lookup` 스킬이 SEC EDGAR API를 지원한다. 사업 내용·재무제표·리스크 요인(Item 1A)의 원천.
- **IPO(신규상장)**: S-1 > S-1/A > FWP > 424B(최종 투자설명서)를 **최신 제출일순**으로 — 문서 우선순위·source_status 규칙은 `.claude/skills/ipo-snapshot/SKILL.md` 참조.
- **재무 데이터**: 웹 검색으로 매출/이익/마진/부채 추이, P/E·P/S·EV/EBITDA 등 밸류에이션 배수. 가능하면 회사 IR 자료와 교차 확인.
- **뉴스·촉매**: WebSearch로 최근 실적 발표, 가이던스, 애널리스트 목표가, 규제·소송 동향.
- **거시**: `database-lookup`의 FRED(금리·물가), Open Targets/FDA(바이오 기업) 등 도메인별 DB.

## 3. 한국 주식 데이터 소스

- **DART 전자공시** (1차, 무료): 사업보고서·분기보고서·반기보고서, 주요사항보고서, 감사보고서. `https://dart.fss.or.kr` 또는 DART OpenAPI(`k-dart` 스킬). 한국 기업 재무·사업의 원천.
- **IPO(신규상장)**: 증권신고서(지분증권) → [기재정정]증권신고서(수요예측 후 공모가 확정) → 투자설명서를 **최신 제출일순**으로. 수요예측 경쟁률·의무보유확약·확정 공모가는 정정 증권신고서가 1차. KRX 상장공시(신규상장 일정), 38커뮤니케이션 등 공모주 포털은 보조(`source_status: "press"`).
- **재무 데이터**: 네이버 증권·KRX 정보데이터시스템 등에서 매출/영업이익/순이익, PER·PBR·배당. 네이버 블로그·증권 페이지가 막히면 `insane-search` 우회.
- **뉴스·촉매**: WebSearch로 실적, 공시, 업종 동향, 애널리스트 리포트 요지.
- **거시**: 한국은행 기준금리, 환율(원/달러), 업종 사이클.

## 4. 접근 차단 시

웹/DB 호출이 402/403/봇차단으로 막히면:
1. `insane-search` 스킬로 우회 (네이버 블로그, X, 차단 사이트 등에 강함)
2. 그래도 안 되면 대체 출처를 찾는다
3. 끝내 못 구하면 해당 항목을 **"데이터 미확보"** 또는 **"검증불가(접근차단)"** 로 명시하고 진행한다. 추측으로 채우지 않는다.

## 5. 출처 인용 규칙 (전 에이전트 공통)

- 모든 수치·사실 주장에는 출처와 가능하면 **시점**을 붙인다: "FY2024 매출 $96B (10-K, 2024-10 제출)".
- 1차 자료(공시) > 회사 IR > 평판 좋은 언론 > 블로그·커뮤니티 순으로 신뢰한다.
- 오래된 데이터를 현재처럼 쓰지 않는다. 데이터 시점을 항상 표기해 팩트체커의 "시점오류" 판정을 피한다.
- 추정·해석은 사실과 분리해 "[추정]" 또는 "[해석]" 태그로 구분한다.
