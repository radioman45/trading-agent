# Trading Agent — 투자 분석 통합 홈

투자 분석 하네스 **3개**가 이 폴더에 산다 — 트레이딩(2차 시장), IPO(신규상장, 미국·한국), 공시 분석(Filings Deep-Read). 의사결정 저널·교훈·캘리브레이션(`decisions/`)은 트레이딩·IPO 하네스 **공용**이다(전체 투자 관점의 단일 학습 루프 — 전 판단 에이전트가 같은 교훈 풀을 읽는다; 공시 하네스는 이해 목적이라 저널을 쓰지 않는다). `decisions/portfolio.json`(보유 SSOT)·`watchlist.md`는 트레이딩 하네스의 세션 간 상태다. 작업공간은 분리: 트레이딩 `_workspace/`, IPO `_workspace_ipo/`, 공시 `_workspace_filings/`. 보고서는 `reports/` 공용(IPO 산출물은 `{회사}_IPO_{날짜}/`). 다른 폴더에서 연 세션은 전역 스킬 `invest-router`가 이 홈으로 라우팅한다. **decisions/ 갱신은 커밋과 한 몸이다**(저널·복기·워치리스트 갱신 직후 `git add decisions/ && git commit` — 학습 코퍼스 무버전 방치 금지).

## 하네스 1: 투자전략 트레이딩 (2차 시장, TradingAgents 구조)

**목표:** 한 종목을 거시 레짐 판정(L0, risk-on/off·유동성·바벨) + 데이터 고정(SSOT) → 분석가 4렌즈 병렬(거시 배경) → Bull/Bear 2라운드 토론 → 리서치 판정(역발상·기회비용) → 거래 계획(R식+꼬리 검산 이중 사이징) → 리스크 3성향 토론 ∥ 포트폴리오 영향 측정(보유 북 집중·동시 꼬리 손실) → 포트폴리오 매니저 최종 게이트(거시·역발상·포트폴리오·기회비용) → 의사결정 저널 기록까지, [TradingAgents](https://github.com/tauricresearch/tradingagents)의 트레이딩 펌 구조 + 뉴로퓨전/월가아재 톱다운 매크로로 자동 수행하고 복기·캘리브레이션으로 학습하는 파이프라인. 능동 모드: 아이디어 스캔(레짐→후보 발굴·워치리스트), 포지션 점검(무효화 트리거 모니터링), 상태 갱신(보유 등록·체결 반영).

**트리거:** 종목 트레이딩/매매/투자전략 분석 요청("{티커} 트레이딩 분석", "사야 돼 팔아야 돼", "거래해도 될까", "거시 레짐/시장 국면", 능동 모드 "뭐 살까/아이디어 스캔/종목 발굴/워치리스트/포지션 점검/보유 등록/체결 반영", 후속 "다시 실행/거시만 다시/토론 한 라운드 더/거래계획만 다시/복기") 시 `trading-strategy` 스킬을 사용하라. 단순 주가 조회·일반 투자 상식 질문은 직접 응답 가능.

**실행 모드:** 서브 에이전트 (파이프라인 + 팬아웃/팬인 + 생성-검증). Bull/Bear R1과 리스크 3성향은 상호 격리(독립 관점 보존), 데이터는 `_workspace/` 파일로만 전달. 계층별 모델 라우팅(저부하 데이터·팟캐스트·해설=sonnet, 분석·생성·판단 코어=opus).

## 하네스 3: 공시 보고서 체계 분석 (Filings Deep-Read)

**목표:** 사용자가 제공한 기업 공시 원문(미국 10-K/10-Q, 한국 DART 분기/반기/사업보고서)을 **외부 호출 없이 제공 파일만으로** 체계적으로 해부해, 매수/매도 판단이 아닌 **'공시·펀더멘털 이해 리포트'**를 산출하는 파이프라인. 아키비스트가 사실을 SSOT로 고정(D0) → 감사관이 원문 대조로 추출 검증(D0.6, 팬아웃 전 게이트) → 4렌즈(재무제표·MD&A·세그먼트·리스크)가 병렬 해부(D1, 상호 격리) → (선택) 산업 자가진단 22질문 심화(D2) → 종합가가 입체 이해 리포트로 통합(D3, 렌즈 간 일치·상충 명시) → 감사관이 종합을 red-team 검증(D3.5) → 입문자용 쉬운 해설판(D4)까지. 트레이딩/IPO 하네스와 달리 **매매 판단·저널 기록을 하지 않는다**(이해가 목적 — 매매 판단은 trading-strategy로). 첫 적용 사례: Micron 10-Q(`sources/IR-Micron/`).

**트리거:** 공시/재무제표 분석 요청("{회사} 공시 분석", "10-K/10-Q 분석", "사업·분기·반기보고서 분석", "DART 보고서 분석", "이 공시 읽어줘", "재무제표 분석", 후속 "SSOT만 다시/특정 렌즈만 다시/산업까지 깊게/종합만 다시/검증만 다시/다른 분기 추가") 시 `filings-analysis` 스킬을 사용하라. 일반 회계·공시 상식 질문은 직접 응답 가능.

**실행 모드:** 서브 에이전트 (파이프라인 + 팬아웃/팬인 + 생성-검증). 4렌즈는 상호 격리(독립 렌즈가 종합의 정보량 원천), 데이터는 `_workspace_filings/` 파일로만 전달. 생성-검증 분리(disclosure-auditor가 격리 감사 — 추출 게이트 + 종합 red-team 2모드). 계층별 라우팅(추출·해부·종합·검증 코어=opus, 해설=sonnet). 산업 22질문은 **선택적 심화 레이어**(기본 off). 산업 자가진단 가이드 방법론은 `industry-diagnostic` 스킬(원본 `assets/industry-guide.md`).
