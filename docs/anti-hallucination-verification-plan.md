# 할루시네이션 방지 검증 계획 (Anti-Hallucination Verification Plan)

작성: 2026-06-24 | 계기: SOL AI반도체TOP2플러스 ETF 분석에서 거시 레짐 에이전트가 **"6/24 코스피 +8.18% 급반등"을 관측 사실로 날조**(작성 시점은 6/24 장 개장 전, 수치도 내부 모순). 이 거짓이 L0에서 17개 산출물 전체로 전파됨.

## 1. 이번 사고의 근본 원인 (5-why)

| # | 질문 | 원인 |
|---|------|------|
| 1 | 왜 거짓 수치가 보고됐나 | macro-strategist가 **아직 열리지 않은 6/24 장의 종가·등락률을 지어냄**(출처도 날조 — sedaily/MBC 인용했으나 해당 데이터 부재) |
| 2 | 왜 안 걸러졌나 | **시간 검증 부재** — "이 날짜·세션이 지금(now) 기준 이미 마감됐는가"를 아무도 확인 안 함 |
| 3 | 왜 수치 모순도 통과했나 | **산술 정합 검증 부재** — 6/23 8,203 → "+8.18%"면 ~8,874여야 하나 8,096.93(오히려 -1.3%)으로 기재, 무검증 |
| 4 | 왜 전체로 퍼졌나 | **L0(거시)는 파이프라인 최상류 배경 레이어** — 하류 12개 에이전트가 배경 사실을 무비판 신뢰, 재검증 게이트 없음 |
| 5 | 왜 오케스트레이터가 못 잡았나 | 의사결정 핵심 사실(특히 "오늘/내일" 날짜·반등)에 대한 **레드플래그 점검 없이** 사용자에게 그대로 중계 |

**핵심 교훈:** ① 시점(temporal) 검증 ② 산술 정합 ③ 출처 실재성 ④ SSOT 규율 ⑤ 상류 오염 차단 게이트 — 다섯 구멍이 동시에 열려 있었다.

## 2. 5계층 방어 (Defense-in-Depth)

### L1 — 예방: 소스 에이전트(특히 macro-strategist·market-data-engineer)의 자기 규율
정량 주장마다 **상태 태그**를 의무화한다: `[관측사실|출처|시각]` / `[추정·가정]` / `[예정·미발생]`. 태그 없는 수치 = 결함.

1. **시점 가드(temporal guard) [HARD]** — 모든 시장 수치는 "{now} 기준 이미 마감된 세션"의 것만 *관측사실*로 쓴다.
   - 한국장(09:00–15:30 KST)·미국장 세션 마감 시각을 now와 대조. **미개장/장중이면 종가·등락률을 절대 단정 금지** → "예정/미확정"으로만.
   - 모든 실행은 Phase 1에서 **현재 시각·요일·각 시장 세션 상태(개장 전/장중/마감)를 명시 고정**하고, 에이전트는 이 앵커 기준으로만 추론.
   - 세션 시차 분리: KST와 미국장(전일 밤) 데이터를 **같은 '오늘'로 섞지 않는다**(이번 SOX +1.7% vs -7.8% 혼선의 원인).

2. **산술 정합 자가검산 [HARD]** — 등락률·수준·시총·비중 등은 항등식으로 자가검증. `new ≈ old×(1+chg)`, 비중 합 ≤100%, PER×E≈P 등. 불일치 시 수치 폐기·재수집.

3. **출처 실재성 [HARD]** — `[출처]`는 **이번 실행에서 실제로 fetch해 본문을 확인한** URL만. 기억·관행 인용 금지("citation theater" 금지). 못 구하면 `미확보(confidence: unavailable)`.

4. **SSOT 규율** — 가격·지수레벨·등락·거래량은 **market-data-engineer 스냅샷이 단일 진실**. macro 등 타 에이전트가 시장 수치를 독자 수집·단정 금지(필요 시 스냅샷 참조 또는 `미확보` 표기). macro의 "스냅샷 비의존" 자유를 **거시 지표(금리·CPI 등)로 한정**하고 지수 종가류는 SSOT로 회수.

### L2 — 탐지: 독립 검증 게이트 (신규 Phase 1.6 "사실 검증")
L0가 전체를 오염시키므로 **분석가 팬아웃(Phase 2) 전에** 격리된 검증자(fact-checker/verifier)가 `00_macro_regime.md` + `00_market_snapshot.json`만 검사:
- ✅ 시점 유효성(미래·미개장 세션 수치 없는지) ✅ 산술 정합 ✅ 출처 실재성(샘플 재fetch) ✅ SSOT 충돌 ✅ "오늘/내일/방금" 날짜 수치 집중 점검
- **⛔ 위반 발견 → 해당 소스 에이전트 1회 재실행 후 통과해야 Phase 2 진행.** (생성-검증 분리 원칙: 작성 세션이 자기 검증 안 함)

### L3 — 봉쇄: 하류 전파 차단
- 하류 에이전트는 배경 수치를 *쓰되 재유도 금지*, 단 **스냅샷과 배경(macro)의 가격·레벨이 충돌하면 ⛔하드 정지**(현 프로토콜의 "상충 출처 병기"를 *가격·핵심 수치*에 한해 정지 게이트로 격상).
- 의사결정 핵심 수치(진입가·손절·손익비·확신도)는 final 단계에서 **출처 추적 가능해야** 함(이번엔 손익비 등은 6/23 종가 SSOT 기반이라 무사 — 이 규율 덕).

### L4 — 오케스트레이터 레드플래그 (내 책임)
핵심 정량 주장 **"클레임 원장"**을 유지하고, 사용자 보고 전 아래는 반드시 자가 점검:
- 🚩 "오늘/어제/내일/방금" 날짜가 붙은 수치 → 세션 마감 여부 확인
- 🚩 라운드넘버 급등/급반등, 결론을 떠받치는 단일 사실 → 교차 확인
- 🚩 "방증한다/증명한다"류 단정 → 그 증거가 관측인지 가정인지 확인
- 검증 불가 시 **사용자에게 "미확인"으로 명시**(중계 시 단정 금지).

### L5 — 학습 루프
이 사고를 `decisions/lessons.md`에 **전체(트레이딩+IPO 공용) 교훈**으로 기록 → 다음 실행에서 전 판단 에이전트가 읽음.

## 4. 우선순위 (비용 대비 효과)

1. **즉효·저비용:** L1 시점 가드 + 산술 정합 + 상태 태그(에이전트 프롬프트 몇 줄) → 이번 사고의 80% 차단.
2. **중간:** L2 검증 게이트(Phase 1.6) — L0 오염 확산을 구조적으로 봉쇄. 토큰 비용 약간↑이나 최상류라 ROI 최고.
3. **상시:** L4 오케스트레이터 레드플래그 — 코드 변경 없이 즉시 적용 가능(내 운영 수칙).
4. **장기:** 가능하면 시점·산술 검증을 **스크립트화**(collect_market_data.py에 세션 마감 검증·정합 assert 추가)해 LLM 판단 의존도↓.
