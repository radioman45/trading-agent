# 모델·추론 강도 선택 가이드 — 학습 노트

> 이 하네스를 실행할 때 어떤 모델과 추론 강도를 쓸지 고민하다 나온 판단을, 다음에 같은 결정을 내릴 때(그리고 다른 하네스에도) 재사용할 수 있게 원리 중심으로 정리한 문서다.

## TL;DR

- **모델: Opus 4.8 유지.** Fable 최고 추론은 이 작업엔 과하다.
- **추론 강도: High 권장** (특히 첫 실분석 같은 고위험·저빈도 실행).
- 비용이 부담되면 중간도 방어 가능 — 단, 무엇을 잃는지 알고 선택할 것(아래 참조).

## 핵심 질문

> "하네스가 잘 돼 있으니 모델·추론 강도를 낮춰도 결과가 비슷하지 않을까?"

→ **절반만 맞다.** 어디에 맞고 어디에 틀린지를 가르는 게 이 문서의 본론이다.

## 원리 1 — 하네스가 보장하는 것 vs 못 메우는 것

하네스(스킬+에이전트 정의+오케스트레이터)가 실제로 하는 일은 **과정의 구조화**다:

| 하네스가 보장하는 것 (구조·검증) | 하네스가 못 메우는 것 (생성·판단) |
|---|---|
| 단계 누락 없음, 실행 순서 | 가장 강한 논거를 **만들어내는** 일 |
| SSOT 일관성(같은 숫자 공유) | 가장 날카로운 반박을 **생성**하는 일 |
| 역할 규율(분석가가 방향 결론 안 냄) | 기계적 중간을 피하고 **판정**하는 일 |
| 체크리스트 검증(손익비≥1.5, 손절≥1.5ATR, 확신도 정의) | 비자명한 리스크를 **발견**하는 일 |
| 격리 구조(독립 관점 보존) | 상충을 **중재**하고 결과론을 **분리**하는 일 |

왼쪽은 규칙으로 못 박을 수 있어 모델·추론 강도에 **거의 무관하게 안정적**이다. 그래서 추론을 낮춰도 *바닥*은 단단하다.

오른쪽은 체크리스트로 못 박을 수 없는 **생성적 추론**이고, 하네스가 *의도적으로 모델에게 남겨둔 빈칸*이다. 여기서 추론 강도 차이가 그대로 드러난다.

## 원리 2 — 하네스는 바닥(floor)을 높이고, 추론은 천장(ceiling)을 정한다

```
품질
 ▲
 │        ┌─────────  ← 천장: 추론 강도가 결정 (통찰의 날카로움)
 │       ╱
 │      ╱
 │  ───┘             ← 바닥: 하네스가 결정 (과정의 신뢰성)
 │
 └──────────────────►
```

- **잘 만든 하네스 + 약한 추론 = 잘 정리된 평범함.** 구조는 완벽한데 통찰이 없는 분석.
- **약한 하네스 + 강한 추론 = 똑똑하지만 들쭉날쭉.** 가끔 빛나지만 단계를 빠뜨리고 일관성이 깨짐.
- 둘은 **대체재가 아니라 보완재**다. 하네스가 좋다고 추론을 깎으면, 깎인 만큼 천장이 내려온다.

## 원리 3 — 이 워크로드의 병목은 모델 천장이 아니다 (그래서 Fable 불필요)

트레이딩 분석의 실제 병목:
1. **데이터 품질** — 뉴스 신선도, 공시 반영, 심리 표본의 대표성
2. **시장의 환원 불가능한 불확실성** — 미래는 모델이 더 똑똑해도 안 보인다

Fable-max와 Opus-high가 내놓는 논거의 차이보다, 위 1·2의 차이가 결과를 훨씬 크게 흔든다. 11개 에이전트 전체에 Fable-max를 씌우는 비용·지연 대비, 실제로 드러나는 품질 우위는 작다. **모델 천장이 병목이 아닐 때 더 비싼 모델은 낭비다.**

→ 반대로, 모델 천장이 진짜 병목인 작업(예: 어려운 알고리즘 설계, 긴 수학적 증명)이라면 Fable이 값을 한다. 작업의 병목이 어디인지 먼저 보는 게 핵심.

## 원리 4 — 그렇다면 아낄 거면 모델이 아니라 "어디서" 아끼나

추론 강도는 **균일하게 줄 필요가 없다.** 인지 부하로 계층을 나누면:

| 계층 | 에이전트 | 인지 부하 | 추론 강도 |
|---|---|---|---|
| 데이터 | market-data-engineer | 낮음(스크립트가 일함) | 중간으로 충분 |
| 분석 | 분석가 4명 | 중간(렌즈 안에서 해석) | 중간~높음 |
| **생성/판단 코어** | bull/bear, research-manager, risk 3성향, portfolio-manager, 복기 | **높음(비구조화 생성)** | **High가 값을 함** |
| 오케스트레이션 | 이 세션(제어 흐름) | 낮음 | 중간으로 충분 |

**가치는 생성/판단 코어에서 나온다.** 적대적 토론·심판·리스크 게이트·복기 — 여기가 하네스가 모델에게 맡긴 빈칸이고, 여기서 High가 보험값을 한다. 저빈도·고위험 작업(가끔 돌리는데 틀리면 실제 돈)이라 추론 강도는 **값싼 보험**이다.

## 적용 — 다음에 이 결정을 내릴 때

1. **이 작업의 병목이 모델 천장인가?** 아니면(데이터/불확실성 바운드) 최상위 모델은 과하다.
2. **가치가 생성되는 곳이 구조화돼 있나, 빈칸인가?** 빈칸(토론·판단·생성)이면 추론 강도를 아끼지 마라.
3. **아낄 거면 균일하게 말고 계층별로.** 기계적·검증 계층에서 아끼고, 생성·판단 계층은 지켜라.
4. **고위험·저빈도면 추론 강도는 보험으로 사라.** 자주 안 돌리는데 틀리면 비싼 작업일수록 High.
